## 核心内容

### 场景设定

这是一节实操课。我们要用 cc4pm 自己的工具，为 cc4pm 构思并规划一个新功能。

这叫做 **"吃自己的狗粮"（Eating Our Own Dog Food）**——用产品本身来改进产品。如果一个产品开发工具连自己都服务不好，怎么说服别人用？

**我们的任务**：为 cc4pm 添加一个"交互式教学系统"——实际上，你正在学习的这套课程（cc4pm-guide）就是这么诞生的。

让我们重走一遍从构思到规划的完整流程。

### Step 1: 探索现状（Plan Mode）

在做任何事之前，先了解现状。这是 Lesson 2 学到的核心原则：**先探索，再规划，再执行。**

```bash
cd cc4pm
claude

# 进入 Plan Mode（按 Shift+Tab 切换两次，或 Esc 后按 Tab）
```

在 Plan Mode 下，让 Claude 帮你了解项目现状：

```
"读一下 .claude/skills/cc4pm-guide/ 的结构，理解目前的教学系统是怎么组织的。
 特别关注：
 1. SKILL.md 的格式和内容结构
 2. lessons/ 目录下有哪些课程
 3. 课程文件的 Markdown 格式规范"
```

**你会观察到**：

Claude 在 Plan Mode 下只读取文件，不做任何修改。它会报告：

- SKILL.md 使用 YAML 前置元数据定义技能名称和描述
- 课程按阶段（stage-1, stage-2...）组织
- 每节课有统一的格式：本课目标 → 核心内容 → 常见问题 → 下一步导航

> 这就是 Lesson 5（CLAUDE.md）中讲的"项目记忆"——Claude 通过读取文件来理解项目上下文。

### Step 2: 头脑风暴（CIS 创意工具）

现状了解清楚了，现在用 CIS 的创意工具来激发想法。

```bash
# 先清理上下文——Step 1 的探索已完成，不需要保留细节
/clear

# 启动头脑风暴
/bmad-brainstorming
```

当 AI 问你主题时，输入：

```
主题：如何让 cc4pm 的教学系统更好？增加什么互动元素？

背景：目前是纯文本的 Markdown 课程，用户在 Claude Code 中阅读。
目标用户是产品主理人和互联网从业者，他们可能不太熟悉命令行。
```

**预期产出**：

AI 会用多种创意技巧帮你生成想法，比如：

```
=== SCAMPER 法 ===
S（替代）：用交互式问答替代被动阅读
C（组合）：课程 + 实操练习 + 即时反馈
A（适应）：根据用户角色（PM/Dev/QA）适配内容
M（修改）：把"课程"变成"任务挑战"
P（另作他用）：把课程笔记变成可复用的 prompt 模板
E（消除）：去掉纯理论，每个概念都附带实操
R（反转）：让用户教 Claude（费曼学习法）

=== 类比思维 ===
如果把教学系统想象成"游戏"...
- 课程进度 = 等级系统
- 完成练习 = 获得经验值
- 掌握技能 = 解锁成就
- 阶段测验 = 关卡 Boss

=== What If 思考 ===
如果用户只有 5 分钟？→ 每课提供"快速版"
如果用户是完全的技术小白？→ 增加基础术语表
如果用户想跳到高级内容？→ 支持灵活导航
```

### Step 3: 需求规划（/plan 命令）

从头脑风暴中选出最有价值的想法，用 `/plan` 转化为结构化的实施方案。

```bash
# 清理上下文——新阶段，新开始
/clear

/plan "为 cc4pm-guide 教学系统添加进度追踪功能：
1. 记录用户完成了哪些课程
2. 显示成就徽章（初学者、入门者、产品达人等）
3. 支持从上次中断的地方继续学习
参考 .claude/skills/cc4pm-guide/SKILL.md 的现有格式"
```

**预期输出**：

```
## 进度追踪功能规划

### 一、功能概述
为 cc4pm-guide 教学 Skill 添加学习进度追踪能力，
让用户能看到自己的学习历程、获得成就激励、
并在多次学习之间保持连续性。

### 二、核心功能
1. 进度记录：标记已完成/未完成的课程
2. 成就系统：完成特定里程碑时显示徽章
3. 断点续学：记住上次学到哪里，下次直接继续

### 三、技术方案
- 进度数据存储在 ~/.claude/ 下的本地文件
- SKILL.md 中的"进度追踪"部分动态更新
- 成就徽章定义在 SKILL.md 的配置中

### 四、实现步骤
- [ ] 定义进度数据格式（JSON）
- [ ] 在每课末尾添加完成标记逻辑
- [ ] 实现成就判断规则
- [ ] 在 SKILL.md 主菜单显示当前进度
- [ ] 添加"继续上次学习"选项
```

### Step 4: 验证方案（使用子代理）

方案制定好了，但它可行吗？是否与现有系统兼容？这正是 Lesson 7 学到的子代理的用武之地——让子代理去做深度验证，不污染我们的主上下文。

```bash
"用子代理检查 /plan 输出的方案：
 1. 这个方案是否与 cc4pm-guide 现有的 SKILL.md 格式兼容？
 2. 进度数据存在 ~/.claude/ 下是否合理？有没有更好的位置？
 3. 有没有遗漏的边界情况？比如多设备同步、数据丢失恢复等"
```

**观察子代理的工作方式**：
- 子代理会在独立上下文中读取 SKILL.md、检查现有格式
- 它可能会读取 scripts/ 目录了解现有的持久化机制
- 最终只返回精炼的评估结果给你

### Step 5: 保存进度

完整的工作流跑完了，别忘了保存——这是 Lesson 2 讲到的会话管理。

```bash
# 给当前会话命名
/rename "cc4pm-guide-进度追踪-规划"

# 保存会话上下文
/save-session
```

下次你可以用 `claude -r "cc4pm-guide"` 恢复这个会话继续工作。

### 跨会话记忆：.tmp 文件模式

在复杂项目中，一个会话往往不够用。当你需要跨多天持续工作时，可以用**会话总结文件**延续上下文：

```bash
# 会话结束前，让 Claude 生成进度总结
"把当前进度保存到 .claude/sessions/day1-summary.md，包含：
 1. 已完成的工作（有证据可验证）
 2. 尝试过但失败的方法（避免重复踩坑）
 3. 下一步计划和待解决问题"

# 第二天，新会话开始时加载
claude
"读取 .claude/sessions/day1-summary.md，从中断处继续"
```

**为什么不直接用 `--continue`？** `--continue` 恢复完整的对话历史，可能包含大量已不相关的探索内容。而 .tmp 总结文件只保留精炼的结论和待办事项，上下文更干净。

**最佳实践**：每个会话一个总结文件（`day1-summary.md`、`day2-summary.md`），不要往同一个文件追加——避免旧上下文污染新工作。
