# Lesson 12: Trigger Map：用户心理到功能映射

## 本课目标

- 理解 Trigger Map 是什么以及为什么它是 WDS 的核心创新
- 掌握 Trigger Map 的 5 步构建过程
- 通过电商产品实例学会完整的 Trigger Map 构建
- 学会用 Trigger Map 做功能优先级排序

## 核心内容

### 为什么 Trigger Map 是 WDS 的灵魂？

先讲一个真实场景——

你的老板说："我们要做一个电商 App，功能包括商品搜索、购物车、订单管理、优惠券、直播带货、社区分享、AR 试穿……"

传统做法：你开始写 PRD，一个功能一个功能列下来，然后问"先做哪个？"——结果就是一场漫长的优先级战争。

**Trigger Map 做法**：先不管"做什么功能"。先问——

> "用户打开这个 App 的时候，脑子里在想什么？心里在怕什么？期待什么？"

这就是 Trigger Map 的核心创新——**从用户心理驱动力（Driving Forces）出发，反向推导功能需求**。

### Trigger Map 的方法论基础

Trigger Map 基于 **Effect Mapping**（效果映射），由 Mijo Balic 和 Ingrid Domingues 发展的方法论，WDS 对其做了两个关键改造：

1. **简化**：去掉了"功能"层——因为功能是从心理驱动力自然推导出来的
2. **增强**：加入了"负面驱动力"——不仅考虑用户"想要什么"，还考虑用户"害怕什么"

### 5 步构建 Trigger Map

整个 Trigger Map 的构建过程分为 5 个步骤：

```
Step 1: 业务目标定义
↓ "我们做这个产品要达到什么业务目标？"

Step 2: 用户角色画像
↓ "谁会使用这个产品？他们有什么特征？"

Step 3: 心理触发因素识别
↓ "这些用户的正面驱动力和负面驱动力是什么？"

Step 4: 功能映射
↓ "要回应这些驱动力，我们需要什么功能？"

Step 5: 评分矩阵
↓ "哪些驱动力最重要？对应的功能该排什么优先级？"
```

### 实操：为"潮品电商 App"构建 Trigger Map

我们用一个"潮品电商 App"的例子，走一遍完整流程。

**启动 Saga**：

```bash
/bmad-agent-wds-saga-analyst
# Saga 会展示欢迎菜单
# 选择 Trigger Map 工作流
```

#### Step 1：定义业务目标

Saga 会先问你："这个产品的核心业务目标是什么？"

好的业务目标应该是**可衡量的**：

```markdown
## 业务目标

### 主要目标
1. 6 个月内达到 10 万月活用户
2. 用户月均下单 2.5 次
3. 客单价 200 元以上

### 次要目标
4. 用户留存率 > 40%（30天）
5. NPS > 50
```

**PM 提醒**：不要写"让用户满意"这种模糊目标。Saga 会要求你给出具体数字——这本身就是在帮你厘清思路。

#### Step 2：定义用户角色

不是简单的"25-35 岁女性"那种人口统计。WDS 的用户角色更关注**心理特征**：

```markdown
## 用户角色

### 角色 1：时尚探索者 Lily
- 核心特征：喜欢发现新品牌和新趋势
- 消费习惯：冲动消费，但对性价比敏感
- 痛点：信息过载，不知道什么真的适合自己
- 目标：在同龄人中保持时尚领先感

### 角色 2：品质追求者 Alex
- 核心特征：注重质量和品牌故事
- 消费习惯：理性消费，愿意为好东西付溢价
- 痛点：怕买到假货或劣质品
- 目标：用合理的价格买到真正有品质的东西

### 角色 3：社交分享者 Mimi
- 核心特征：消费行为受社交影响大
- 消费习惯：看别人推荐才买，爱分享购物体验
- 痛点：缺乏可信赖的推荐来源
- 目标：获得社交认同和分享的快乐
```

#### Step 3：识别心理触发因素（核心中的核心）

这一步是 Trigger Map 最精华的部分。Saga 会引导你为每个用户角色识别两种驱动力：

**正面驱动力**（用户被什么吸引）：

```markdown
### 时尚探索者 Lily 的正面驱动力
- DF+1: 发现感（"哇，这个好酷，我从没见过"）
- DF+2: 领先感（"我是第一个穿这个的"）
- DF+3: 省力感（"不用我自己翻遍全网了"）
```

**负面驱动力**（用户害怕什么）：

```markdown
### 时尚探索者 Lily 的负面驱动力
- DF-1: 后悔焦虑（"万一买了不合适怎么办"）
- DF-2: 信息焦虑（"选择太多了，我该选哪个"）
- DF-3: 落后恐惧（"别人都在穿新的了，我还在穿旧的"）
```

**为什么要关注"负面驱动力"？**

因为用户不行动的原因往往是恐惧，而不是缺乏吸引力。你的产品不仅要"吸引用户来"，还要"消除用户不来的理由"。

#### Step 4：从驱动力映射到功能

现在，每个功能的存在都有了明确的理由：

```markdown
### 功能映射

| 驱动力 | 对应功能 | 解释 |
|--------|---------|------|
| DF+1 发现感 | 个性化推荐流 | 帮用户发现他们不知道但会喜欢的东西 |
| DF+2 领先感 | 新品首发预告 | 让用户第一时间知道新品 |
| DF+3 省力感 | AI 穿搭推荐 | 不用自己搭配，AI帮你 |
| DF-1 后悔焦虑 | 无忧退换 + 真人评价 | 降低购买心理门槛 |
| DF-2 信息焦虑 | 策展式分类 | 不用面对 10000 个商品，只看精选 |
| DF-3 落后恐惧 | 趋势提醒 | 推送当季流行趋势 |
```

看到了吗？每个功能旁边都有一个"为什么做这个功能"的理由，而且这个理由不是"竞品有"或"老板要求"，而是**用户的心理需求**。

#### Step 5：评分矩阵（优先级排序）

最后，用评分矩阵来决定先做什么：

```markdown
### 评分矩阵

| 驱动力 | 影响用户数 | 驱动力强度 | 实现复杂度 | 综合优先级 |
|--------|----------|----------|----------|----------|
| DF+1 发现感 | 高（3个角色） | 高 | 中 | P0 |
| DF-1 后悔焦虑 | 高（3个角色） | 高 | 低 | P0 |
| DF+3 省力感 | 中（2个角色） | 高 | 高 | P1 |
| DF-2 信息焦虑 | 中（2个角色） | 中 | 中 | P1 |
| DF+2 领先感 | 低（1个角色） | 高 | 低 | P2 |
| DF-3 落后恐惧 | 低（1个角色） | 中 | 低 | P2 |
```

**PM 的关键收获**：优先级不再是拍脑袋了。"发现感"影响所有 3 个角色且驱动力强度高，所以是 P0；"领先感"只影响 1 个角色，所以是 P2。这个逻辑任何利益相关方都能理解。

### Trigger Map 的输出文件

整个过程完成后，Saga 会生成以下文件：

```
{output_folder}/
├── B-Trigger-Map/
│   └── trigger-map.md          # 完整的 Trigger Map 文档
├── A-Product-Brief/
│   └── product-brief.md        # 产品简报（如果是从简报开始的）
└── _progress/
    └── 00-design-log.md        # 设计日志，记录所有决策
```

`trigger-map.md` 的结构通常包含：

```markdown
# Trigger Map: [产品名]

## Business Goals（业务目标）
...

## User Roles（用户角色）
...

## Driving Forces Map（驱动力映射）
### Role 1: [角色名]
#### Positive Driving Forces
#### Negative Driving Forces

### Role 2: [角色名]
...

## Priority Matrix（优先级矩阵）
...

## Key Insights（关键洞察）
...
```

### 三种工作模式的选择

Saga 在做 Trigger Map 时提供三种模式，选哪个取决于你的情况：

| 你的情况 | 推荐模式 | 理由 |
|---------|---------|------|
| 第一次做，想学方法 | Workshop（研讨会） | Saga 一步步引导，你深度参与每个步骤 |
| 有竞品分析/用户研究 | Suggest（建议） | Saga 先从你的文档提取信息，给建议，你审核 |
| 时间紧张，信任 AI | Dream（AI自主） | Saga 自主完成，你最后审核和调整 |

**PM 建议**：第一次一定要用 Workshop 模式。这不只是为了产出 Trigger Map，更是为了让你**内化这套思维方式**。之后你可以切到 Suggest 或 Dream 来提速。

## 常见问题

**Q: Trigger Map 和用户故事地图（User Story Mapping）有什么区别？**

A: 用户故事地图关注"用户做什么"（行为），Trigger Map 关注"用户为什么做"（心理）。两者互补——你可以先用 Trigger Map 找到心理驱动力，再用故事地图把驱动力拆成具体的用户故事。实际上，BMM 的 Epics/Stories 正好扮演了故事地图的角色。

**Q: 负面驱动力为什么重要？竞品分析不就够了吗？**

A: 竞品分析告诉你"别人做了什么"，但不告诉你"用户为什么不来"。负面驱动力专门回答后者。比如"支付安全顾虑"是很多电商用户不下单的隐性原因，但你做竞品分析很难发现——因为所有竞品都在强调"安全支付"，你以为这是标配，实际上它是一个需要被专门设计的"消除恐惧"方案。

**Q: 驱动力太多了怎么办？**

A: 正常。一个产品通常会有 15-30 个驱动力。用评分矩阵来筛选——优先关注"影响用户数高 + 驱动力强度高"的前 6-8 个。其他的放到后续迭代中。Saga 会帮你做这个筛选。

**Q: 已有产品也能用 Trigger Map 吗？**

A: 非常适合。选择"从已有文档提取"模式，让 Saga 读取你现有的 PRD、用户研究报告、数据分析报告，从中提取心理驱动力。然后对比你现有的功能和驱动力的匹配度——你可能会发现有些功能没有对应的驱动力（考虑删掉），有些驱动力没有被功能覆盖（考虑新增）。
