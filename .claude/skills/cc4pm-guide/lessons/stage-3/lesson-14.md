# Lesson 14: 故事讲述：产品叙事与演示

## 本课目标

- 认识 CIS 故事讲述大师（索菲亚）的能力
- 掌握 25 种故事框架的分类和选择方法

- 实操创建一个完整的产品发布故事

## 核心内容

### 产品经理最被低估的能力：讲故事

你可能做了一份完美的 PRD、一个无懈可击的 Trigger Map、一套精密的架构设计——但如果你不能用一个好故事把这些"卖"出去，一切都白搭。

真实场景：

```
场景 1：向老板汇报
老板：这个新功能怎么样？
你（没有故事技能）：这个功能用了微服务架构，覆盖了3个用户角色的5个驱动力……
老板：（打哈欠）所以呢？

你（有故事技能）：想象一下小李，一个90后宝妈，每天花2小时比价。
我们的功能让她10分钟搞定——省下的时间她能给孩子多讲一个故事。
上线第一周就有3000个"小李"在用。
老板：好！什么时候上线？
```

**故事 = 影响力的放大器。**

### 认识索菲亚：CIS 故事讲述大师

cc4pm 的 CIS 模块中有一个专门的故事讲述工作流，背后的 AI 角色叫**索菲亚（Sophia）**——一位大师级叙事引导者。

**索菲亚的能力**：

- 25 种故事框架，覆盖商业叙事的所有场景
- 10 步工作流，从故事目标到最终交付
- 多种模式：她可以引导你写，也可以替你写
- 自动保存和版本管理

**启动索菲亚**：

```bash
/bmad-cis-storytelling
```

### 25 种故事框架一览

索菲亚提供的 25 种故事框架分为 5 大类：

#### 1. 转变类叙事（Transformation）—— 5种

| 框架 | 一句话描述 | 最适合 |
|------|----------|-------|
| 英雄之旅 | 经典的冒险 → 考验 → 蜕变弧线 | 创始人故事、品牌传奇 |
| Pixar 故事骨架 | "从前...直到有一天..." 的情感递进 | 产品发布、用户案例 |
| 客户旅程故事 | 用户从痛苦到满足的转变 | 案例研究、营销内容 |
| 挑战克服弧线 | 遭遇障碍 → 突破 → 胜利 | 困难项目的复盘汇报 |
| 角色成长弧线 | 个人从 A 状态成长到 B 状态 | 团队激励、文化建设 |

#### 2. 战略类叙事（Strategic）—— 5种

| 框架 | 一句话描述 | 最适合 |
|------|----------|-------|
| 品牌故事 | 品牌价值观和使命 | 品牌建设、对外宣传 |
| 愿景叙事 | 描绘未来美好图景 | 战略汇报、团队动员 |
| 起源故事 | "为什么我们做这件事" | 公司简介、融资 |
| 定位故事 | 为什么我们不一样 | 竞争分析、市场策略 |
| 文化故事 | 组织价值观和行为准则 | 内部文化建设 |

#### 3. 说服类叙事（Persuasive）—— 5种

| 框架 | 一句话描述 | 最适合 |
|------|----------|-------|
| 路演叙事 | 问题 → 方案 → 机会 → 行动号召 | 投资人路演 |
| 销售故事 | 从痛点到解决方案的价值呈现 | 客户提案 |
| 变革故事 | 为什么必须改变、怎么改变 | 组织变革、流程优化 |
| 筹款故事 | 连接价值观和影响力 | 公益、社会企业 |
| 倡导故事 | 号召支持某个运动或政策 | 行业布道、政策推动 |

#### 4. 分析类叙事（Analytical）—— 5种

| 框架 | 一句话描述 | 最适合 |
|------|----------|-------|
| 数据叙事 | 把数据变成有说服力的故事 | 数据分析报告、BI汇报 |
| 案例研究 | 完整记录一个项目的过程和结果 | 客户案例、项目复盘 |
| 研究叙事 | 把研究发现变成易懂的故事 | 用研报告、市场调研 |
| 洞察叙事 | 揭示一个反直觉的发现 | 产品洞察、策略建议 |
| 过程故事 | 展示"幕后故事"——怎么做到的 | 技术博客、团队分享 |

#### 5. 情感类叙事（Emotional）—— 5种

| 框架 | 一句话描述 | 最适合 |
|------|----------|-------|
| 钩子驱动 | 用震撼开头抓住注意力 | 演讲开场、社交媒体 |
| 冲突-解决 | 核心矛盾的建立和化解 | 戏剧性产品故事 |
| 共情故事 | 让受众站在他人角度思考 | 用户研究分享、设计决策 |
| 人情味故事 | 聚焦个体的真实经历 | 内容营销、PR |
| 脆弱故事 | 真诚分享失败和挣扎 | 创始人分享、团队信任 |

### 故事框架选择指南

面对 25 种框架，怎么选？索菲亚会根据你的场景自动推荐，但你也可以用这个速查表：

```
你的场景是什么？
│
├── 对外汇报
│   ├── 给投资人 → 路演叙事（Pitch Narrative）
│   ├── 给客户 → 客户旅程故事 / 销售故事
│   └── 给媒体 → 品牌故事 / 起源故事
│
├── 对内沟通
│   ├── 给老板 → 数据叙事 / 洞察叙事
│   ├── 给团队 → 愿景叙事 / 变革故事
│   └── 给新员工 → 文化故事 / 起源故事
│
├── 产品相关
│   ├── 产品发布 → Pixar 故事骨架 / 钩子驱动
│   ├── 功能介绍 → 客户旅程故事 / 挑战克服弧线
│   └── 用研分享 → 共情故事 / 研究叙事
│
└── 数据相关
    ├── 数据报告 → 数据叙事
    ├── 项目复盘 → 案例研究 / 过程故事
    └── 策略建议 → 洞察叙事
```

### 索菲亚的 10 步工作流

启动索菲亚后，她会带你走 10 步：

```
Step 1: 故事上下文设置
↓ "你要讲什么故事？给谁听？想传达什么？"

Step 2: 选择故事框架
↓ 基于上下文推荐最佳框架

Step 3: 收集故事元素
↓ 用苏格拉底式提问引出你的故事素材

Step 4: 设计情感弧线
↓ "开头观众该感受什么？高潮呢？结尾呢？"

Step 5: 打造开头钩子
↓ 创建一个让人无法忽视的开头

Step 6: 撰写核心叙事
↓ 你可以选择自己写、AI 写、或合作写

Step 7: 创建多版本
↓ 短版（社交媒体）、中版（邮件）、长版（演讲）

Step 8: 使用指南
↓ 哪个渠道用哪个版本

Step 9: 精炼和打磨
↓ 最后的调整

Step 10: 生成最终文档
↓ 保存完整的故事文档
```

### 实操：为产品发布创建故事

**场景**：你的"潮品电商 App"即将上线，需要一个内部发布故事来激励团队。

```bash
/bmad-cis-storytelling
```

**Step 1：设置上下文**

```
索菲亚：让我们开始吧。先告诉我——

1. 这个故事的目的是什么？
   → 内部团队动员，为产品上线打气

2. 你的目标受众是谁？
   → 30人的产品+开发+设计团队

3. 你想传达什么核心信息？
   → 我们做了一件有意义的事，而且做得很好

4. 有什么限制条件吗？
   → 5分钟演讲，需要有情感共鸣
```

**Step 2：索菲亚推荐框架**

```
索菲亚：基于你的目的（团队动员）和受众（内部团队），
我推荐"Pixar 故事骨架"，因为：
- 它有天然的情感递进结构
- 适合5分钟以内的短叙事
- "直到有一天..."的转折点很适合产品发布

要用这个框架吗？还是你更偏好其他的？
```

**Step 3-5：收集素材和设计弧线**

索菲亚会通过一系列问题引出你的故事素材：

```
索菲亚：
- 从前，你们团队面对的市场是什么样的？
- 每天，用户在经历什么痛苦？
- 直到有一天，什么契机让你们决定做这件事？
- 因为这样，你们做了什么决定？
- 又因为这样，遇到了什么挑战？
- 直到最终，你们交付了什么？
```

**Step 6：生成核心叙事**

```
索菲亚的初稿：

---

从前，有一群年轻人——他们每天花两个小时在各种 App 之间
切换，只为找到一件不撞衫的 T 恤。

每天，他们被淹没在 10 万件商品里，看了 500 张图却还是
不知道该选哪一件。他们不缺选择——他们缺的是"被理解"。

直到有一天，我们问了一个不一样的问题：
"如果 App 不是给你推荐10000件商品，而是只推荐10件
——但每一件都像是专门为你挑的呢？"

因为这个问题，我们重新设计了整个推荐逻辑。
不是"别人买了什么"，而是"你是什么风格的人"。

因为这样，我们做了上千次的算法调优、
200 轮的用户测试、37 次的 UI 大改。

直到最终，我们做出了这个——
一个不是让你"逛"的 App，而是让你"发现自己"的 App。

今天，它上线了。
而你——在座的每一个人——就是让它成为现实的人。

---
```

**Step 7：多版本输出**

```
短版（社交媒体）：
"如果 App 不推荐10000件商品，而是只推荐10件——
但每一件都像专门为你挑的呢？
今天，我们上线了。"

中版（全员邮件）：
"大家好，今天有个好消息……"（2段版本）

长版（演讲稿）：
完整5分钟演讲稿
```

## 常见问题

**Q: 25 种框架是不是太多了？常用的是哪几种？**

A: 对 PM 来说，最常用的 5 种是：(1) 路演叙事——给老板/投资人汇报；(2) 数据叙事——季度报告/数据分析；(3) 客户旅程故事——案例分享/营销；(4) Pixar 故事骨架——产品发布/内部动员；(5) 洞察叙事——策略建议/需求评审。掌握这 5 种就能覆盖 80% 的场景。

**Q: 索菲亚生成的故事可以直接用吗？**

A: 可以作为很好的初稿。但好故事需要你的"真实感"——你的个人经历、团队的真实挑战、具体的数字和细节。建议的流程是：索菲亚生成框架和初稿 → 你填入真实素材 → 索菲亚帮你打磨语言和节奏。

**Q: 不擅长写作的 PM 能用好这个工具吗？**

A: 这恰恰是索菲亚最大的价值。她不是要你"写"——她是用苏格拉底式提问引导你"说"。你只需要回答她的问题，她会把你的回答组织成一个有结构、有情感、有说服力的故事。你不需要写作天赋，你需要的是对产品的热情和对用户的理解——这些你都有。

**Q: 故事讲述和数据汇报冲突吗？**

A: 完全不冲突。好的数据汇报就是数据叙事——"上个月转化率从 3% 提升到 5%"是数据，"上个月，我们帮 2000 个用户找到了他们要的东西——比上个月多了 600 人"是故事。同样的数据，不同的讲法，完全不同的说服力。
