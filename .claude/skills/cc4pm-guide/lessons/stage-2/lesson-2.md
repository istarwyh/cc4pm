### 重点框架详解

#### 1. Jobs-to-be-Done（JTBD）——最被低估的 PM 利器

JTBD 的核心思想：用户不是在"买产品"，而是在"雇佣产品完成某个任务"。

**经典案例**：

> "人们买电钻，不是因为他们想要电钻，而是因为他们想要墙上的孔。"
>
> 更深一层：他们想要挂上一幅画。再深一层：他们想要家里更温馨。

**JTBD 的 5 个核心问题**：

```
1. 用户在"雇佣"你的产品完成什么任务？
2. 他们想取得什么进展（Progress）？
3. 他们目前用什么替代方案（Alternatives）？
4. 现有方案有什么不满意的地方？
5. 什么情况下他们会"解雇"你的产品？
```

#### 2. 蓝海战略——让竞争变得无关紧要

蓝海战略的核心是"四步动作框架"：

```
┌─────────┬────────────────────────────────┐
│  消除    │ 行业里习以为常但其实没必要的因素 │
├─────────┼────────────────────────────────┤
│  减少    │ 被过度设计的因素               │
├─────────┼────────────────────────────────┤
│  提升    │ 行业标准以上需要大幅提升的因素   │
├─────────┼────────────────────────────────┤
│  创造    │ 行业从来没有提供过的全新因素     │
└─────────┴────────────────────────────────┘
```

**示例**——用蓝海战略分析"AI 记账工具"：

- **消除**：复杂的手工录入界面、专业会计术语
- **减少**：功能复杂度（80% 的中小企业只用 20% 的功能）
- **提升**：自动化程度（扫描发票自动归类）、数据可视化
- **创造**：AI 财务顾问（主动建议省钱方案）、行业对标分析

#### 3. 商业模式画布——9 个模块讲清楚怎么赚钱

```
┌──────────────┬──────────────┬──────────────┬──────────────┬──────────────┐
│   关键合作    │   关键活动    │              │   客户关系    │              │
│              │              │   价值主张    │              │   客户细分    │
│   关键资源    │              │              │   渠道        │              │
├──────────────┴──────────────┤              ├──────────────┴──────────────┤
│          成本结构            │              │          收入来源            │
└─────────────────────────────┴──────────────┴─────────────────────────────┘
```

填写顺序建议：客户细分 -> 价值主张 -> 渠道 -> 客户关系 -> 收入来源 -> 关键资源 -> 关键活动 -> 关键合作 -> 成本结构

#### 4. 价值主张画布——确保产品和市场匹配

价值主张画布是商业模式画布的"放大镜"，聚焦两个核心模块的匹配：

```
用户画像（右侧）：              价值图（左侧）：
┌─────────────────┐            ┌─────────────────┐
│ 用户任务（Jobs）  │  ← 匹配 → │ 产品和服务        │
│ 用户痛点（Pains） │  ← 匹配 → │ 止痛药            │
│ 用户收益（Gains） │  ← 匹配 → │ 收益创造器        │
└─────────────────┘            └─────────────────┘
```

**步骤 3：Victor 引导你逐步完成 9 个环节**

Victor 的创新策略工作流是一个完整的 9 步流程：

```
Step 1: 建立战略背景
       ↓ 你描述公司情况、市场压力、当前商业模式
Step 2: 分析市场格局与竞争态势
       ↓ Victor 选择 2-4 个市场分析框架
Step 3: 分析当前商业模式
       ↓ Victor 用商业模式画布等工具解剖你的现状
Step 4: 识别颠覆机会
       ↓ 用 JTBD、蓝海战略等找到战略空白
Step 5: 生成创新机会
       ↓ 产出 5-10 个具体的创新机会
Step 6: 开发和评估战略选项
       ↓ 综合成 3 个战略方向，多维度评估
Step 7: 推荐战略方向
       ↓ 大胆推荐 + 清晰论证
Step 8: 构建执行路线图
       ↓ 三阶段实施计划
Step 9: 定义指标和风险缓解
       ↓ 领先指标、滞后指标、决策关卡
```

每个步骤完成后，Victor 会保存当前进度，并给你选项：

```
[a] 高级引导 —— 更深入地探讨
[c] 继续 —— 进入下一步
[p] 派对模式 —— 召集虚拟团队一起讨论
[y] YOLO —— 让 Victor 自己跑完剩余步骤
```

**步骤 4：获得完整的创新策略报告**

输出是一份结构化的创新策略文档，包括：

- 公司名称和战略焦点

- 三阶段执行路线图

### 什么是 BMM？为什么现在要提它？

前两课（L11-L12）你一直在用 CIS 的工具做**发散**——生成创意、分析蓝海。但创意不落地就是空谈。**接下来需要一套"收敛"系统把创意变成可交付的产品。**

这就是 **BMM（BMad Method，BMad 方法论）**——cc4pm 四大模块中负责**产品落地全流程**的核心引擎。

```
┌────────────────────────────────────────────────────────┐
│  BMM（BMad Method）— 从想法到交付的完整流水线             │
├────────────────────────────────────────────────────────┤
│                                                          │
│  BMM 不是一个工具，而是一组 AI 代理组成的"虚拟产品团队"：    │
│                                                          │
│  分析师（Analyst）  → 市场研究、竞品分析、领域调研          │
│  PM John           → PRD 创建、需求验证、需求编辑          │
│  架构师（Architect）→ 技术方案、架构决策                    │
│  SM Bob            → 冲刺规划、进度追踪、路线纠正           │
│  QA                → 测试策略、E2E 测试、质量保障           │
│  开发者（Dev）      → 编码实现、代码审查                    │
│                                                          │
│  每个代理有自己的性格、专长和启动命令。                      │
│  你是产品负责人，他们是你的团队。                            │
│                                                          │
└────────────────────────────────────────────────────────┘
```

**CIS 和 BMM 的关系**：

| | CIS（创意智能套件） | BMM（BMad 方法论） |
|---|---|---|
| **职责** | 想清楚**做什么** | 把它**做出来** |
| **阶段** | 发散：创意→策略→方向 | 收敛：需求→拆解→交付 |
| **代理** | Carson、Victor 等 6 个创意专家 | 分析师、PM、SM 等 6 个执行角色 |
| **产出** | 创意列表、创新策略报告 | PRD、Epics、Stories、冲刺计划 |
| **本阶段课程** | L11 头脑风暴、L12 创新策略 | L13 市场研究、L14 PRD、L15 拆解、L16 冲刺 |

**类比理解**：CIS 像是"研发实验室"——探索方向、验证假设；BMM 像是"产品工厂"——把验证过的方向变成可执行的产品计划。

从下一课（L13）开始，你将正式接触 BMM 的代理。但衔接从这一课就开始了——Victor 的创新策略报告是 BMM 流程的**输入素材**。

### 从 CIS 到 BMM：创新产出如何连接 PRD

理解了 BMM 后，再看这条完整的产品工作流就清晰了：

```
[CIS] 头脑风暴 → 100+ 想法 → 筛选出 3-5 个方向
                    ↓
[CIS] 创新策略  → 蓝海分析 → 选定 1 个战略方向
                    ↓
[BMM] 分析师    → 市场研究 → 验证市场机会（Lesson 13）
                    ↓
[BMM] PM John   → 创建 PRD → 将战略变成产品需求（Lesson 14）
                    ↓
[BMM] PM John   → Epics/Stories → 拆解成可执行任务（Lesson 15）
                    ↓
[BMM] SM Bob    → 冲刺规划 → 开始开发（Lesson 16）
```

**关键衔接动作**：

1. 把 Victor 输出的创新策略报告保存到项目的 `planning-artifacts` 目录
2. 启动 BMM 的 PM 代理时，告诉 John："请参考我们的创新策略报告来创建 PRD"
3. John 会自动读取并整合这些材料

## 常见问题

**Q: CIS 产出的报告质量不高怎么办？还能交给 BMM 用吗？**

A: 可以。BMM 的 PM 代理（John）不会盲目照搬 CIS 的报告——他会在创建 PRD 时追问你细节、挑战假设。CIS 的报告是"起点素材"而非"最终决策"。哪怕报告粗糙，也比从零开始给 John 讲需求要高效得多。

**Q: Victor 的分析是否可靠？**

A: Victor 提供的是结构化的思考框架和分析过程，而非市场预测。他会帮你系统地思考问题，但最终的判断需要你结合实际市场数据来做。如果在创新策略工作流中需要真实的市场数据，建议先用 BMM 的分析师代理做一次市场研究。

**Q: 30 个框架太多了，我该从哪个开始？**

A: 你不需要自己选框架。告诉 Victor 你的情况，他会根据你公司的阶段（初创/成熟）、行业特点、战略目标自动推荐最合适的 2-4 个框架。对大多数 PM 来说，掌握 JTBD + 蓝海战略 + 商业模式画布就能覆盖 80% 的场景。

**Q: 一次创新策略分析需要多长时间？**

A: 取决于你的深入程度。如果你在每个步骤都选择 [c] 继续，大约 30-60 分钟就能走完。如果你选择 [a] 高级引导深入讨论，可能需要 2-3 小时。建议第一次完整走一遍，之后可以根据需要跳到特定步骤。
