### 三种研究工作流

Mary 提供三种研究工作流，各有侧重。你可以根据需要选择一种或组合使用。

#### 工作流 1：市场研究（MR）

**适用场景**：你想了解一个市场的全貌——用户在哪里、竞争对手是谁、有没有机会。

**必要条件**：需要联网搜索能力（Web Search）。如果 Claude Code 没有联网功能，Mary 会提示你。

**MR 的 6 步流程**：

```
Step 1: 初始化
       ↓ 确认研究主题和范围
Step 2: 客户行为分析
       ↓ 深入了解目标用户的行为模式
Step 3: 客户痛点分析
       ↓ 系统梳理用户的核心痛点
Step 4: 客户决策分析
       ↓ 了解用户是怎么做购买/使用决策的
Step 5: 竞争分析
       ↓ 全面的竞争格局扫描
Step 6: 研究综合报告
       ↓ 汇总所有发现，输出结论和建议
```

**输出成果**：一份完整的市场研究报告，保存在 `planning-artifacts/research/` 目录下。

#### 工作流 2：领域研究（DR）

**适用场景**：你要进入一个不熟悉的行业（比如医疗、金融、教育），需要快速建立领域知识。

**DR 的 6 步流程**：

```
Step 1: 初始化
       ↓ 确认要研究的领域
Step 2: 领域分析
       ↓ 行业结构、关键参与者、商业模式
Step 3: 竞争格局
       ↓ 行业内的竞争态势和格局变化
Step 4: 监管焦点
       ↓ 法规、合规要求、政策趋势
Step 5: 技术趋势
       ↓ 正在改变这个行业的技术力量
Step 6: 研究综合报告
       ↓ 汇总报告
```

**领域研究 vs 市场研究的区别**：

| 维度 | 市场研究（MR） | 领域研究（DR） |
|------|--------------|--------------|
| 焦点 | 用户和竞争 | 行业和知识 |
| 产出 | "这个市场值不值得做" | "这个行业是怎么运作的" |
| 适合时机 | 有明确产品想法，需要验证 | 刚进入新行业，需要建立认知 |
| 深度 | 用户行为、购买决策 | 行业结构、监管、技术 |

#### 工作流 3：技术研究（TR）

**适用场景**：你需要了解某种技术的可行性和实现路径，比如"用 AI 做发票识别靠不靠谱"。

**TR 的 6 步流程**：

```
Step 1: 初始化
       ↓ 确认技术研究主题
Step 2: 技术概览
       ↓ 技术现状、成熟度、主流方案
Step 3: 集成模式
       ↓ 怎么跟现有系统集成
Step 4: 架构模式
       ↓ 可选的技术架构方案
Step 5: 实现研究
       ↓ 具体的实现路径和成本评估
Step 6: 研究综合报告
       ↓ 技术可行性结论
```

### 竞争分析框架和模板

不管你用哪种研究工作流，竞争分析都是核心环节。以下是 Mary 常用的竞争分析框架。

#### 波特五力分析

用来评估行业的整体竞争强度：

```
           ┌───────────────────┐
           │  新进入者的威胁    │
           │  （门槛高低？）    │
           └─────────┬─────────┘
                     │
┌───────────────┐    │    ┌───────────────┐
│  供应商议价权  │────┼────│  买方议价权    │
│ （谁更强势？） │    │    │ （用户选择多？）│
└───────────────┘    │    └───────────────┘
                     │
           ┌─────────┴─────────┐
           │  替代品的威胁      │
           │ （用户能用别的？）  │
           └───────────────────┘

           中心：现有竞争者的竞争强度
```

#### 竞品对标矩阵

```markdown
| 维度 | 我们 | 竞品 A | 竞品 B | 竞品 C |
|------|------|--------|--------|--------|
| 核心功能 | | | | |
| 定价 | | | | |
| 目标用户 | | | | |
| 技术优势 | | | | |
| 用户体验 | | | | |
| 市场份额 | | | | |
| 薄弱环节 | | | | |
```

#### SWOT 分析

```
┌─────────────────────┬─────────────────────┐
│      优势 (S)        │      劣势 (W)        │
│  我们做得好的地方    │  我们做得不好的地方  │
│                     │                      │
├─────────────────────┼──────────────────────┤
│      机会 (O)        │      威胁 (T)        │
│  外部环境中的机会    │  外部环境中的风险    │
│                     │                      │
└─────────────────────┴──────────────────────┘
```

### 市场机会评估方法

当 Mary 帮你完成研究后，你需要评估发现的机会值不值得投入。这里有一个实用的评估框架：

**机会评估矩阵**：

| 评估维度 | 权重 | 评分（1-5） | 加权得分 |
|---------|------|------------|---------|
| 市场规模（TAM） | 20% | | |
| 增长速度 | 15% | | |
| 竞争强度（越小越好） | 15% | | |
| 用户痛点程度 | 20% | | |
| 我们的能力匹配 | 15% | | |
| 商业化可行性 | 15% | | |
| **加权总分** | **100%** | | |

- 4.0+ 分：强烈建议投入
- 3.0-4.0 分：值得深入研究
- 2.0-3.0 分：需要谨慎评估
- 2.0 以下：建议放弃

### 如何将研究结果转化为产品决策

研究不是目的，决策才是。以下是从研究报告到产品决策的转化方法：

```
研究发现                →  产品决策
───────────────────────────────────────
"目标用户集中在25-35岁"  →  界面设计偏年轻化
"竞品都忽略了移动端"     →  移动优先策略
"用户最大痛点是数据导出"  →  数据导出作为核心功能
"市场年增长30%"         →  值得加大投入
"监管趋严"             →  合规能力作为差异化
```

**决策输出模板**：

```markdown
## 基于市场研究的产品决策

### 关键发现
1. [发现1]：[数据/证据]
2. [发现2]：[数据/证据]
3. [发现3]：[数据/证据]

### 产品决策
1. [决策1]：基于[发现X]，我们决定...
2. [决策2]：基于[发现Y]，我们决定...

### 优先级排序
| 优先级 | 决策 | 预期影响 | 实施难度 |
|--------|------|---------|---------|
| P0     |      |         |         |
| P1     |      |         |         |
| P2     |      |         |         |

### 后续行动
- [ ] 行动 1
- [ ] 行动 2
```

### 研究工作流的组合使用

在实际工作中，三种研究工作流通常需要组合使用：

| 项目阶段 | 推荐研究组合 | 理由 |
|---------|------------|------|
| 新进入一个行业 | DR → MR → TR | 先了解行业，再看市场，最后确认技术 |
| 现有产品寻找新方向 | MR → TR | 先看市场机会，再评估技术可行性 |
| 评估技术趋势 | TR → MR | 先看技术成熟度，再看市场需求 |
| 竞品分析 | MR（聚焦竞争分析步骤） | 重点跑 Step 5 |

## 常见问题

**Q: 市场研究需要联网搜索，但我的环境没有网络访问怎么办？**

A: Mary 的市场研究工作流确实需要 Web Search 能力来获取最新数据。如果没有联网能力，Mary 会在启动时提示你。你有两个替代方案：(1) 自己收集数据后以文件形式提供给 Mary 分析；(2) 使用领域研究（DR）工作流，它更多依赖 AI 的知识库而非实时搜索。

**Q: Mary 的市场数据准确吗？**

A: Mary 通过联网搜索获取数据，但她不是专业数据库。对于关键决策依据（如市场规模数字），建议交叉验证：(1) 查阅行业报告（如艾瑞、TalkingData）；(2) 向行业专家确认；(3) 看公开的上市公司财报。Mary 的最大价值是帮你建立分析框架和发现方向，而不是提供精确数字。

**Q: 研究报告太长了，怎么提取关键信息？**

A: Mary 的报告有清晰的章节结构。对 PM 最有用的是"关键发现"和"建议"两个部分。如果你想要更精简的版本，直接告诉 Mary："请帮我提取报告中的 Top 5 关键发现和 3 个核心建议。"

**Q: 产品简报和 PRD 是什么关系？**

A: 产品简报（Product Brief）是"电梯演讲"——一页纸讲清楚产品是什么、为谁做、为什么做。PRD 是"详细蓝图"——完整的需求文档。通常先做简报确定方向，再做 PRD 展开细节。在 cc4pm 中，Mary 做简报（[CB]），John 做 PRD（[CP]）。
