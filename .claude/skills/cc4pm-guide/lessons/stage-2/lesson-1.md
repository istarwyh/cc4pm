### 认识头脑风暴教练：Carson

Carson 是 CIS 模块中的头脑风暴教练代理，定位是"精英头脑风暴专家"（Elite Brainstorming Specialist）。他的性格像一个充满激情的即兴表演教练——能量爆棚，用 "YES AND" 不断叠加你的想法，疯狂鼓励野性思维。

**启动 Carson**：

```bash
/bmad-brainstorming
```

Carson 会用你配置的语言（中文）跟你打招呼，然后展示菜单：

```
[MH] 重新显示菜单
[CH] 与代理聊天
[BS] 引导我进行任何主题的头脑风暴
[PM] 启动派对模式
[DA] 结束对话
```

选择 **[BS]** 就可以开始一次完整的头脑风暴。

**Carson 的核心理念**：

- 心理安全感是突破性创新的前提——没有"蠢问题"
- 今天的疯狂想法就是明天的创新产品
- 幽默和玩耍是严肃的创新工具

### PM 必须掌握的 5 大技巧

从 36 种技巧中，以下 5 种对产品主理人最实用，建议优先掌握。

#### 1. SCAMPER 法——产品迭代的瑞士军刀

SCAMPER 是 7 个英文单词的首字母，每个字母代表一种改造动作：

```
S - Substitute（替代）：用什么可以替代现有方案？
C - Combine（组合）：能和什么合并？
A - Adapt（适应）：能从其他领域借鉴什么？
M - Modify/Magnify（修改/放大）：放大或缩小某个方面？
P - Put to other use（另作他用）：还能用在什么地方？
E - Eliminate（消除）：去掉什么？
R - Reverse/Rearrange（逆转/重排）：反过来会怎样？
```

**实操示例**——用 SCAMPER 改造"在线问卷工具"：

| 动作 | 提问 | 创意产出 |
|------|------|---------|
| S 替代 | 用 AI 对话替代传统表单？ | 对话式问卷、语音问卷 |
| C 组合 | 问卷 + 数据分析一体化？ | 自带 BI 的问卷平台 |
| A 适应 | 借鉴游戏化设计？ | 答题闯关式问卷 |
| M 放大 | 把"填写"变成"共创"？ | 协作式需求收集工具 |
| P 另作他用 | 不做调研，做员工反馈？ | 匿名反馈 + 情绪追踪 |
| E 消除 | 去掉"提交"按钮？ | 实时同步、渐进式保存 |
| R 逆转 | 让被调研者来设计问卷？ | 用户自助需求提交平台 |

#### 2. 六顶思考帽——决策前的多角度扫描

每顶帽子代表一种思考模式，逼自己从不同角度看问题：

```
白帽（数据）：有哪些客观事实和数据？
红帽（直觉）：你的第一感觉是什么？
黑帽（谨慎）：最大的风险和问题是什么？
黄帽（乐观）：最大的好处和价值是什么？
绿帽（创意）：还有没有其他可能？
蓝帽（管理）：思考过程本身有没有遗漏？
```

**PM 使用场景**：在做产品决策前，依次戴上六顶帽子，确保不遗漏任何维度。比如评估"要不要做社交功能"：

- 白帽：日活 10 万，用户平均使用时长 5 分钟，竞品 A 有社交功能但使用率仅 3%
- 红帽：直觉告诉我用户并不需要在这个工具里社交
- 黑帽：社交功能开发成本高、审核风险大、可能分散核心功能注意力
- 黄帽：如果做好了，能大幅提升用户粘性和分享裂变
- 绿帽：不一定做"社交"，可以做"协作"——两个人共同编辑文档
- 蓝帽：我们是不是被"社交"这个词限制了？核心诉求可能是"让多人协同使用"

#### 3. 第一性原理——回到本质重新思考

抛开所有既定假设，回到最基本的事实，从零开始推导。

**思考步骤**：

```
1. 明确问题："用户为什么需要这个功能？"
2. 剥离假设："去掉所有'大家都这么做'的惯性思维"
3. 回到事实："用户的真实需求是什么？"
4. 重新推导："从这个真实需求出发，最好的解决方案是什么？"
```

**经典案例**——马斯克造火箭：

- 传统思维：火箭很贵，因为"一直都很贵"
- 第一性原理：火箭的原材料成本只占售价的 2%，贵的是一次性使用
- 突破方案：做可回收火箭，把成本降低 10 倍

**PM 实操**——重新思考"用户注册"：

- 传统思维：用户必须注册才能使用
- 第一性原理：我们真正需要的是"识别用户"，不是"收集信息"
- 突破方案：免注册使用，设备指纹识别，在用户需要时才引导注册

#### 4. What If 假设法——探索极端可能性

通过极端假设打开思路，常用句式是"如果...会怎样？"

**实用模板**：

```
- "如果这个功能免费会怎样？"
- "如果用户数量突然变成 100 倍会怎样？"
- "如果竞争对手明天就做了同样的功能会怎样？"
- "如果我们的目标用户完全变了会怎样？"
- "如果这个功能必须在 1 天内上线会怎样？"
- "如果完全由 AI 来做会怎样？"
```

#### 5. 类比思维——从其他领域偷师

把一个领域的解决方案移植到另一个领域。

**PM 常用类比来源**：

| 来源领域 | 类比思路 | 产品启发 |
|---------|---------|---------|
| 游戏 | 等级系统、成就解锁 | 用户成长体系 |
| 餐饮 | 自助餐 vs 套餐 vs 点菜 | 定价模式设计 |
| 医疗 | 预防 > 治疗 | 从修复 Bug 到预防 Bug |
| 教育 | 因材施教 | 个性化推荐 |
| 物流 | 最后一公里配送 | 用户体验的最后一步 |

### 实操演练：一次完整的产品创意风暴

现在让我们用 Carson 做一次真实的头脑风暴。

**场景**：为一个面向中小企业的 AI 记账工具探索差异化功能。

**步骤 1：启动 Carson**

```bash
/bmad-brainstorming
```

**步骤 2：选择 [BS] 开始头脑风暴**

```
你：BS

Carson：太棒了！让我们开始一场创意风暴吧！
请告诉我：
1. 你想探索什么主题？
2. 有什么背景信息我应该知道的？
3. 你期望什么样的产出？
```

**步骤 3：描述你的主题**

```
你：我想为一个面向中小企业的 AI 记账工具探索差异化功能。
目前市场上的记账工具都很同质化，我们想找到独特的价值主张。
目标是产出 100+ 个功能创意。
```

**步骤 4：Carson 引导你走过多种技巧**

Carson 会根据你的主题，智能选择适合的技巧组合，比如：

```
Carson：YES！我爱这个挑战！AI 记账工具？让我们把它变成
中小企业的"财务超级英雄"！

首先，让我们用 SCAMPER 来热身...
[S] 如果用 AI 对话替代所有手动录入？
[C] 如果把记账 + 发票 + 报税 + 经营分析全部合一？
...

接下来，让我们用类比思维——如果记账工具是一个"私人健身教练"...
- 每日"财务打卡"
- "财务体检报告"
- "不健康支出预警"
...

现在用 What If 打开脑洞...
- 如果这个记账工具能预测未来 3 个月的现金流？
- 如果它能自动帮中小企业找到省税方案？
...
```

**步骤 5：获得结构化输出**

头脑风暴结束后，你会得到：

| 输出部分 | 内容 | 你该怎么用 |
|---------|------|-----------|
| 想法列表 | 100+ 条创意，按类别分组 | 浏览、标记感兴趣的 |
| 可行性初评 | 每个想法的初步评估 | 快速筛选优先级 |
| 聚合主题 | 把相似想法归类 | 发现趋势和方向 |
| 下一步建议 | 推荐接下来做什么 | 决定下一个动作 |

### 技巧组合策略：如何选择

不同场景适合不同的技巧组合：

| 场景 | 推荐组合 | 原因 |
|------|---------|------|
| 新产品从 0 到 1 | 第一性原理 + What If + 类比思维 | 需要跳出框架，从本质出发 |
| 现有产品迭代 | SCAMPER + 属性列举 + 用户旅程 | 围绕现有产品系统改进 |
| 竞品差异化 | 六顶思考帽 + 逆向思维 + 类比 | 需要多角度审视和反向启发 |
| 用户痛点挖掘 | 五个为什么 + 角色扮演 + 故事板 | 深入理解用户，发现隐性需求 |
| 团队共创 | 脑力接龙 + 圆桌轮转 + 画廊漫步 | 确保每个人的声音被听到 |
| 打破僵局 | 最坏情况法 + 随机刺激 + 疯狂八分钟 | 用"反常"方式激活思维 |

### 100+ 想法的分类与筛选

当你有了 100+ 个想法后，别慌。按照这个流程筛选：

```
100+ 想法
    │
    ▼
[第一轮] 去重合并 → 60-80 个独立想法
    │
    ▼
[第二轮] 可行性快筛 → 30-40 个可行想法
    │  标准：技术可行 + 市场有需求 + 资源可支持
    ▼
[第三轮] 价值评估 → 10-15 个高价值想法
    │  标准：用户价值 x 商业价值 x 实现难度
    ▼
[第四轮] MVP 筛选 → 3-5 个 MVP 候选
    │  标准：最小投入 + 最快验证 + 最大学习
    ▼
下一步：进入创新策略分析（Lesson 6）
```

**Pro Tip**：筛选时不要一个人做决定。把 30-40 个可行想法带到团队讨论中，用"名义小组"或"决策矩阵"来集体决策。

## 常见问题

**Q: 36 种技巧我记不住怎么办？**

A: 完全不需要记住。Carson 会根据你的主题自动选择合适的技巧。你只需要在 Carson 建议的技巧中跟着走就行。如果你想指定某个技巧，直接告诉 Carson "我想用 SCAMPER 法" 就好。

**Q: 头脑风暴的想法质量不高怎么办？**

A: 头脑风暴的原则是"先量后质"。100 个想法里有 5 个好的就值了。质量筛选是第二步的事情。在风暴阶段，不要评判任何想法，包括你觉得"很蠢"的——有时候最蠢的想法稍微一改就变成了最好的方案。

**Q: 一个人也能头脑风暴吗？**

A: 当然！Carson 就是你的"虚拟团队"。一个人和 Carson 对话，效果不亚于一个小团队的头脑风暴。Carson 会从不同角度挑战你的思维，弥补单人思考的盲区。

**Q: 头脑风暴的结果怎么和后续工作流衔接？**

A: 头脑风暴产出的想法列表可以直接输入到下一课要学的"创新策略分析"中，进行蓝海分析和商业模式设计。也可以作为创建 PRD 的输入素材。整个 BMM + CIS 工作流是无缝衔接的。
