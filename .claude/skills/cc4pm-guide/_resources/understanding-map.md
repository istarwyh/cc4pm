## 使用场景

### 场景 1：新产品从0到1（PM主导）
**涉及模块**：CIS → BMM → WDS
**流程**：
1. CIS 头脑风暴 → 生成100+想法
2. CIS 创新策略 → 蓝海分析、商业模式
3. BMM 市场研究 → 竞争分析、用户需求
4. WDS 项目简报 → 愿景、定位、成功标准
5. WDS Trigger Map → 用户心理映射
6. BMM 创建PRD → 完整需求文档
7. BMM 创建架构 → 技术方案
8. WDS UX场景 → 用户旅程
9. BMM 冲刺规划 → 开发计划
10. 工程工具链 → 开发、测试、审查

### 场景 2：已有产品优化（PM+工程协作）
**涉及模块**：CIS → BMM
**流程**：

2. CIS 问题解决 → 根本原因定位

## 方法论与最佳实践

### 核心设计哲学

1. **以用户心理为驱动**（WDS Trigger Map）：功能设计从"为什么"开始
2. **AI First Product Development**：AI 代理作为团队成员参与全流程
3. **质量内建**：从 PRD 到代码，每个阶段都有验证
4. **持续学习**：从每次会话中提取知识，演进为技能
5. **微文件架构**：每个工作流步骤自包含，按需加载
