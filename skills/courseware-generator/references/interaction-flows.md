```markdown

- 基础操作：掌握日常使用的基本功能

- 方法论：理解最佳实践和设计哲学

- 进阶用户：约 X5 分钟

### 欢迎界面的关键要素

| 要素 | 说明 | 示例 |
|------|------|------|
| 项目名称 | 醒目的标题 | `# Claude Code 插件集合（cc4pm）交互式教学` |
| 一句话介绍 | 快速说明项目是什么 | `cc4pm 是一个为工程师提供生产就绪的 Claude Code 插件集合` |

| 主菜单 | 4 个清晰的选项 | `[1] 从头开始` / `[2] 跳到特定阶段` / `[3] 查看进度` / `[4] 退出` |

### 单课结构模板

```markdown
# Lesson X: [课程标题]

## 本课目标
- 目标 1：具体可衡量的学习目标
- 目标 2：具体可衡量的学习目标

## 核心内容

### 概念讲解
[用简洁、口语化的语言讲清楚"是什么"和"为什么"]

### 实战示例
[提供可直接运行的代码/命令]

```bash
# 示例命令
/example-command --option value
```

### 进阶内容（可选）
[针对进阶用户的额外说明]

## 常见问题

**Q: [问题 1]？**

A: [答案 1]

**Q: [问题 2]？**

A: [答案 2]

## 下一步

- [1] 进入下一课：[Lesson X+1: 下一课标题]
- [2] 返回主菜单
- [3] 退出学习
```

### 课程学习的交互流程

```
用户选择进入课程

    ↓

显示课程内容
├─ 课程标题
├─ 本课目标（2-3 个）
├─ 核心内容
│  ├─ 概念讲解
│  ├─ 实战示例
│  └─ 进阶内容（可选）
├─ 常见问题（2-4 个）
└─ 下一步选项（3 个）

    ↓

    ↓

├─ [1] 进入下一课 → 验证前置条件 → 显示下一课
├─ [2] 返回主菜单 → 显示欢迎界面
└─ [3] 退出学习 → 显示退出界面
```

### 课程示例

```markdown
# Lesson 2: 探索项目结构

## 本课目标

- 理解 cc4pm 的目录结构
- 了解各个目录的职责
- 能够快速定位所需文件

## 核心内容

### 理解项目结构

cc4pm 的目录结构设计遵循清晰的职责划分：

```
cc4pm/
├── agents/         # 专业化子代理
├── skills/         # 工作流定义和领域知识
├── commands/       # 斜杠命令
├── hooks/          # 触发式自动化
├── rules/          # 始终遵循的准则
├── mcp-configs/    # MCP 服务器配置
├── scripts/        # 跨平台 Node.js 工具
└── tests/          # 测试套件
```

### 快速定位文件

假设你想了解 `/tdd` 命令是如何工作的：

1. 先看 `commands/` 目录
2. 找到 `tdd.md`
3. 查看它调用了哪些 agents

```bash
# 1. 查看 commands 目录
ls commands/

# 2. 查看 tdd 命令文件
cat commands/tdd.md
```

## 常见问题

**Q: 为什么分这么多目录？**

A: 清晰的目录结构帮助用户快速定位所需内容，同时便于维护和扩展。

**Q: 我可以在项目中添加新的目录吗？**

A: 建议遵循现有结构，但可以根据需要添加子目录。

## 下一步

- [1] 进入下一课：[Lesson 3: 运行第一个命令]
- [2] 返回主菜单
- [3] 退出学习
```

| 标记 | 含义 | 使用场景 |
|------|------|----------|
| ✅ 已完成 | 课程已完成 | 用户已学习过这门课 |
| ⬜ 待学习 | 当前可学习 | 前置课程已完成 |
| 🔒 锁定 | 需要先完成前置阶段 | 前置课程未完成 |

## 交互设计最佳实践

### 清晰的选择

每个交互点都应该是清晰的：

| 问题 | 改进前 | 改进后 |
|------|--------|--------|
| 菜单不清晰 | "请选择下一步" | "[1] 进入下一课 [2] 返回主菜单 [3] 退出" |
| 状态不明 | "阶段2" | "阶段 2：核心功能 🔒 待解锁（需先完成阶段 1）" |
| 缺少引导 | "请输入答案" | "请输入你的答案（1-4），或输入 '帮助' 获取提示" |

### 即时反馈

用户操作后应该立即获得反馈：

```
用户选择 [1] 进入下一课

    ✅ 已进入 Lesson 3：探索项目结构
```

```
用户选择 [2] 跳到特定阶段

    ⚠️ 无法跳转 - 请先完成阶段 1
```

### 一致的导航

在整个教学过程中保持一致的导航：

- 始终显示"返回主菜单"选项
- 每个课程都显示"下一步"选项
- 进度页面始终可访问

### 错误处理

优雅地处理错误输入：

```
请输入 1-4 的数字，或输入 '帮助'：

> 5

❌ 无效选择，请重新输入：

> help

可用选项：
[1] 进入下一课
[2] 返回主菜单
[3] 退出学习

请输入 1-3：
```
