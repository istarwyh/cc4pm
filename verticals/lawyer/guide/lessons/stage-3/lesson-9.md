# Supervisor 工作流：Detect → Execute → Evaluate → Package

> 本课程用于讲解 Claude Code 垂直行业工作台方法，不提供法律行业解决方案。Supervisor 演示的是任务调度方法，不是法律项目管理标准。

## 本课目标

学习如何用 Supervisor 把目录、规则、Skills、Agents、Reviewer 和交付包串成一条接近实际的课程工作流。

## 本课要讲的 Claude Code 功能

- Supervisor 作为流程调度者。
- Detect：识别任务、材料缺口和执行顺序。
- Execute：调用合适的 Skill 或 Agent。
- Evaluate：独立复核输出质量。
- Package：整理客户可读交付包和复盘资产。

## 为什么适合垂直行业知识工作

垂直行业项目需要流程控制：先做什么、谁来做、做到什么程度、何时复核、如何交付。Supervisor 的课程价值是把这些流程显性化，而不是让模型凭感觉推进。

## 核心概念

流程编排、任务发现、执行分派、质量评估、交付打包、复盘沉淀

## 本案中的演示任务

围绕 SaaS 合同纠纷教学案例，设计一条课程化流程：读取项目目录 → 建立材料索引 → 运行示例 Skill → 分派示例 Agent → 复核输出 → 形成课堂交付包 → 记录可复用模板。

## 推荐提示词

```text
请为本课程案例设计一个 Supervisor 教学流程，按 Detect → Execute → Evaluate → Package 展开。重点说明每一步要讲的 Claude Code 能力、需要读取的项目文件、产出的课堂材料，以及哪些地方要由学员替换成自己的 SOP。
```

## 预期输出

一份 Supervisor 课程流程计划，包含每一步的目标、输入、执行者、输出、复核点和交付包位置。

## 学员如何替换成自己的 SOP

要求学员把四段流程改成自己机构真实项目的流程，例如“接收材料 → 初步整理 → 专业审查 → 合伙人复核 → 客户汇报 → 模板归档”。

## 课堂练习

让学员用自己的项目写一版 Detect → Execute → Evaluate → Package 流程图，并标注每一步适合 Skill、Agent、Rule 还是人工复核。

## 检查点 / 常见误区

- 是否把 Supervisor 当成万能执行者，而不是调度者。
- 是否缺少 Evaluate 环节。
- 是否客户交付包里混入内部过程或提示词痕迹。
- 是否流程无法映射到真实团队分工。

## 讲课提示

本课是全课程的串联点，要让学员看到 Claude Code 功能不是孤立的：目录、规则、Skill、Agent、Reviewer 都需要被流程编排起来。

## 本课交付物

一份 Supervisor 课程工作流计划。
