# Agent Team：用法律案例讲角色拆分与协作边界

> 本课程用于讲解 Claude Code 垂直行业工作台方法，不提供法律行业解决方案。`agents/` 中的角色是教学模板，不是官方法律 Agent Team。

## 本课目标

学习如何把一个复杂行业项目拆成多个 Agent 角色，并让每个角色只负责自己边界内的任务。

## 本课要讲的 Claude Code 功能

- Agent 角色定义。
- 多角色任务拆分。
- 输入输出边界。
- 复核角色与生成角色分离。

## 为什么适合垂直行业知识工作

真实项目往往不是一个人从头做到尾，而是事实整理、专业分析、客户沟通、质量复核分工协作。Agent Team 的课程价值是教行业团队如何把自己的组织分工映射到 Claude Code，而不是由 cc4pm 替行业定义标准团队。

## 核心概念

角色边界、任务分派、并行协作、复核隔离、责任可见、专业分工外置

## 本案中的演示任务

使用 `verticals/lawyer/agents/` 作为课堂样例，拆解 intake、合同审查、诉讼准备、风险复核等角色。重点讲每个 Agent 的职责边界、输入材料和交付物，而不是讲这些角色应当如何作出法律判断。

## 推荐提示词

```text
请把 verticals/lawyer/agents 里的角色当作教学模板，分析这个 Agent Team 如何体现“事实整理、专业分析、交付准备、独立复核”的分工。请指出哪些角色定义必须由真实律师团队按自己的组织方式替换。
```

## 预期输出

Agent Team 分工表，包含角色目标、输入材料、输出物、禁止事项、交接关系和需要替换的机构 SOP。

## 学员如何替换成自己的 SOP

要求学员按自己团队真实分工重命名角色，例如主办律师、协办律师、实习律师、合规复核、客户经理。课程只提供拆分方法，不规定角色配置。

## 课堂练习

让学员把自己的一个项目拆成 3–5 个角色，写清每个角色“只做什么、不做什么、输出什么、谁来复核”。

## 检查点 / 常见误区

- 是否让一个 Agent 包办所有工作。
- 是否让复核 Agent 同时写正文。
- 是否没有定义输入输出边界。
- 是否把教学角色名称当成律所标准岗位。

## 讲课提示

把 Agent Team 讲成“组织协作的工程化表达”，不要讲成“cc4pm 提供了一套法律机器人团队”。

## 本课交付物

一份 Agent Team 角色拆分图，以及本机构角色替换清单。
