# 上下文管理：让 Claude Code 按任务理解材料

> 本课程用于讲解 Claude Code 垂直行业工作台方法，不提供法律行业解决方案。上下文管理讲的是材料组织方法，不是法律事实认定。

## 本课目标

学习如何把大量行业材料拆成可处理的上下文批次，先建立材料索引，再按任务读取材料，避免一次性倾倒所有信息。

## 本课要讲的 Claude Code 功能

- 文件读取与材料索引。
- 上下文窗口管理。
- 按任务边界控制输入材料。
- 用来源标注保持输出可追溯。

## 为什么适合垂直行业知识工作

垂直行业项目常常材料多、版本多、命名不统一。直接把所有材料塞给模型，会让任务边界变模糊。分批建立上下文，可以让 Claude Code 先知道“有哪些材料”，再决定“本轮任务需要哪些材料”。

## 核心概念

上下文窗口、材料索引、任务分批、来源标注、先事实后判断

## 本案中的演示任务

按合同、补充协议、邮件、聊天记录、付款记录、系统记录分批读取模拟材料。第一轮只做材料索引，不做法律分析；第二轮才进入具体任务。

## 推荐提示词

```text
请先只读取本项目的材料清单和文件名，生成材料索引、建议读取顺序和每类材料可能服务的课程任务。不要开始法律分析，也不要生成结论。
```

## 预期输出

材料索引、优先读取顺序、任务与材料映射表、待确认材料列表。

## 学员如何替换成自己的 SOP

要求学员把“材料索引字段”改成本机构常用字段，例如材料名称、来源人、取得时间、是否脱敏、对应任务、是否可给客户看。

## 课堂练习

给学员一组模拟材料文件名，让他们先写读取计划，再让 Claude Code 生成材料索引。对比“先索引”与“直接分析”的输出差异。

## 检查点 / 常见误区

- 是否第一轮就开始做法律结论。
- 是否没有记录材料来源。
- 是否把所有材料一次性倒入上下文。
- 是否没有说明本轮任务边界。

## 讲课提示

本课要让学员看到：Claude Code 的上下文能力必须配合项目文件和任务边界，才能服务真实工作流。

## 本课交付物

案件材料索引与分批读取计划。
