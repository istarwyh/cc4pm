# CLAUDE.md：把机构工作守则写进项目

> 本课程用于讲解 Claude Code 垂直行业工作台方法，不提供法律行业解决方案。`CLAUDE.md` 是项目协作规则，不是法律意见模板。

## 本课目标

理解 `CLAUDE.md` 如何把项目角色、保密边界、输出格式、复核要求和禁止事项写成 Claude Code 可持续遵守的工作守则。

## 本课要讲的 Claude Code 功能

- 项目级记忆和规则注入。
- 在项目内固定输出风格、边界和复核要求。
- 把“每次都要提醒 AI 的话”沉淀为项目文件。

## 为什么适合垂直行业知识工作

垂直行业有大量隐性规则：什么不能说、什么必须标来源、什么必须人工复核、客户版本不能出现什么。`CLAUDE.md` 把这些规则前置，减少每次对话重复解释。

## 核心概念

项目记忆、工作边界、保密要求、输出口径、复核节点、禁止事项

## 本案中的演示任务

把 `templates/CLAUDE.md` 复制到案件教学工作台根目录，并改成适合本课程演示的项目规则：强调教学用途、来源标注、敏感信息保护、客户版本清理和律师复核。

## 推荐提示词

```text
请基于本课程目标，生成一版项目 CLAUDE.md。它要约束 Claude Code 如何处理材料、如何标注来源、如何区分课堂示例和正式法律意见、如何提醒人工复核。不要写成法律 SOP。
```

## 预期输出

项目级 `CLAUDE.md`，包含角色、边界、保密、输出格式、禁止事项、复核要求和课程用途声明。

## 学员如何替换成自己的 SOP

要求学员把课堂模板中的“示例规则”替换成机构真实规则，例如本机构的客户沟通口径、文书复核流程、敏感信息处理标准、内部文件命名方式。

## 课堂练习

让学员写出自己机构最常重复提醒 AI 的 10 条要求，再分成“必须遵守”“输出格式”“禁止事项”“需要人工复核”四类，放入 `CLAUDE.md`。

## 检查点 / 常见误区

- 是否把 `CLAUDE.md` 写成了长篇法律知识库。
- 是否缺少敏感信息和客户版本边界。
- 是否没有区分课堂示例与正式交付。
- 是否仍然依赖每次手动提醒，而不是项目规则。

## 讲课提示

强调 `CLAUDE.md` 的价值是稳定协作，不是替代专业判断。它承载的是机构工作守则。
