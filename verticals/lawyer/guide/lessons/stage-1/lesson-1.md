# cc4Lawyer 全景：不是法律解决方案，而是垂直课程

> 本课程用于讲解 Claude Code 垂直行业工作台方法，不提供法律行业解决方案。案例、模板、Skill、Agent 和 Rule 都是教学脚手架，真实使用时必须由律师团队替换为自己的 SOP 和复核标准。

## 本课目标

建立正确定位：cc4Lawyer 不是提示词课，也不是 cc4pm 交付的法律 AI 系统，而是一门用法律场景讲 Claude Code 能力组合的垂直行业课程。

## 本课要讲的 Claude Code 功能

- 项目目录如何承载长期上下文。
- `CLAUDE.md` 如何承载项目级工作守则。
- Skills 如何把重复任务封装成流程。
- Agents 如何把团队角色拆成协作单元。
- Supervisor / Reviewer 如何把执行、复核和交付分开。

## 为什么适合垂直行业知识工作

垂直行业的难点通常不是“问一个好问题”，而是材料多、流程长、多人协作、输出要可追溯。Claude Code 的价值在于把这些工作组织成项目，而不是把每次对话当成孤立聊天。

## 核心概念

垂直课程、行业工作台、案例驱动教学、专业 SOP 替换、可复核输出、模板沉淀

## 本案中的演示任务

用 SaaS 软件服务合同纠纷案例串联一条接近实际的教学流程：客户材料进入项目目录 → 写入项目规则 → 分批建立上下文 → 设计示例 Skills → 拆分示例 Agents → 由 Supervisor 串联任务 → Reviewer/Evaluator 复核 → 形成课堂交付包。

## 推荐提示词

```text
请把这个 SaaS 合同纠纷案例当作教学场景，说明 Claude Code 工作台课程应如何串联项目目录、CLAUDE.md、上下文管理、Skills、Agents、Supervisor、Reviewer 和交付包。注意：不要输出法律结论，只讲课程结构和能力点。
```

## 预期输出

一张课程总览图或总览笔记，展示“Claude Code 能力点 → 法律场景演示 → 学员替换自己的 SOP”的主线。

## 学员如何替换成自己的 SOP

要求学员列出自己机构真实存在的流程资产，例如接案清单、合同审查表、证据目录模板、客户汇报模板、复核清单。课堂只教如何把这些资产工程化，不替他们定义专业内容。

## 课堂练习

让学员选一个自己熟悉的业务场景，写下三列：已有材料、重复流程、最终交付物。再把它映射到 Claude Code 的目录、规则、Skill、Agent、Reviewer。

## 检查点 / 常见误区

- 检查是否把“示例流程”误读成“标准法律 SOP”。
- 检查是否仍在收集提示词，而不是设计项目工作台。
- 检查是否能说清每个 Claude Code 能力解决的课程问题。

## 讲课提示

反复强调：我们教的是搭建方法，不是法律判断。法律案例只是让课程更接近真实工作流。
