# 案件初始化 Skill：用示例讲清流程封装

> 本课程用于讲解 Claude Code 垂直行业工作台方法，不提供法律行业解决方案。`lawyer-case-intake` 是教学模板，用来演示 Skill 结构，不是标准接案 SOP。

## 本课目标

学习如何把一个高频、可重复、边界清晰的任务设计成 Skill，并用案件初始化作为课堂示例。

## 本课要讲的 Claude Code 功能

- Skill 的触发场景、输入材料、输出结构和边界声明。
- 如何把“每次都要做的流程”从临时提示词沉淀为可复用能力。
- 如何在 Skill 中写清楚不确定事项和人工复核点。

## 为什么适合垂直行业知识工作

行业专家每天会重复做很多结构相似的工作：接案、访谈、审查、归档、汇报。Skill 的意义不是让 cc4pm 提供行业标准，而是教专家把自己的标准流程写成 Claude Code 能执行的结构。

## 核心概念

Skill 设计、输入边界、输出 schema、来源标注、缺口识别、人工确认

## 本案中的演示任务

使用 `verticals/lawyer/skills/lawyer-case-intake/` 作为课堂样例，讲解一个“案件初始化” Skill 应如何声明适用场景、需要读取哪些材料、输出哪些字段、哪些内容必须标注为待确认。

## 推荐提示词

```text
请不要直接替我写法律结论。请分析 lawyer-case-intake 这个教学 Skill 的结构：它的输入是什么、输出是什么、边界在哪里、哪些部分应该由真实律师团队替换成自己的接案 SOP。
```

## 预期输出

Skill 结构拆解表，包含触发场景、输入材料、输出字段、复核点、可替换 SOP 区域。

## 学员如何替换成自己的 SOP

要求学员把示例中的案件摘要、时间线、材料缺口、追问清单，替换成自己机构真实接案流程中的字段和问法。课程不规定这些字段的法律专业内容。

## 课堂练习

让学员写一版自己的“接案初始化 Skill 设计草图”：什么时候用、读取哪些材料、输出哪些表格、哪些地方必须人工确认。

## 检查点 / 常见误区

- 是否把示例 Skill 当成标准法律接案 SOP。
- 是否只写提示词，没有定义输入输出结构。
- 是否缺少“无法确认时如何标注”。
- 是否没有写清哪些内容必须由专业人员复核。

## 讲课提示

把重点放在 Skill 的工程结构：触发、材料、步骤、输出、边界。法律字段只是案例载体。

## 本课交付物

一份案件初始化 Skill 设计草图，以及本机构需要替换的 SOP 字段清单。
