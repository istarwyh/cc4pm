# 证据目录 Skill：教会学员对齐事实、材料与交付格式

> 本课程用于讲解 Claude Code 垂直行业工作台方法，不提供法律行业解决方案。`lawyer-evidence-catalog` 是教学模板，不是法院或律所的标准证据目录规则。

## 本课目标

学习如何把“材料 → 事实 → 交付表格”的对齐过程封装为 Skill，让输出更容易复核和交付。

## 本课要讲的 Claude Code 功能

- 如何让 Skill 从多个文件提取材料线索。
- 如何把材料与事实主张、证明目的、风险备注放进同一张表。
- 如何把“缺口”和“待补强事项”写成后续任务，而不是隐藏在正文里。

## 为什么适合垂直行业知识工作

垂直行业大量交付物都是结构化清单：证据目录、尽调清单、审计底稿、合规检查表。Claude Code 可以帮助把材料和表格字段对齐，但字段含义和判断标准必须由行业专家定义。

## 核心概念

材料对齐、结构化清单、证明目的、缺口追踪、交付格式、复核路径

## 本案中的演示任务

使用 `verticals/lawyer/skills/lawyer-evidence-catalog/` 作为课堂样例，把合同、邮件、微信、发票、流水、登录记录映射到一个证据目录表。重点讲表格结构和来源追踪，不讲法律证明力结论。

## 推荐提示词

```text
请把 lawyer-evidence-catalog 当作教学 Skill 来拆解：它如何把多份材料整理成一张可复核表格？哪些列是课程演示结构，哪些列需要律师团队按自己的标准重写？
```

## 预期输出

证据目录 Skill 的结构说明、字段替换建议、材料到表格的映射路径、后续补材料任务列表。

## 学员如何替换成自己的 SOP

要求学员把示例表头替换成自己机构交付物的表头，例如证据目录、尽调问题清单、合规风险台账或客户材料清单。

## 课堂练习

给学员 6 个模拟材料名称，让他们设计一张“材料对齐表”。要求每一行都能回到来源文件，并明确哪些字段必须人工确认。

## 检查点 / 常见误区

- 是否只生成漂亮表格但无法追溯来源。
- 是否把材料摘要写成确定法律事实。
- 是否没有把缺口转成后续任务。
- 是否没有说明表头来自本机构 SOP。

## 讲课提示

本课重点是“结构化交付物如何从项目材料中长出来”，不是证明责任或诉讼策略教学。

## 本课交付物

一份材料对齐 Skill 设计稿，以及本机构交付表格字段清单。
