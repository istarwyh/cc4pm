# 合同审查 Skill：把专业表格变成可复用课程动作

> 本课程用于讲解 Claude Code 垂直行业工作台方法，不提供法律行业解决方案。`lawyer-contract-review` 是教学模板，不是权威合同审查标准。

## 本课目标

学习如何把行业专家已有的审查表、风险分级和业务解释口径，设计成可复用 Skill。

## 本课要讲的 Claude Code 功能

- Skill 如何引用项目材料和指定输出表格。
- 如何在提示中固定“先确认立场、再整理材料、最后输出表格”的流程。
- 如何让 Skill 输出便于复核，而不是直接给出不可追溯结论。

## 为什么适合垂直行业知识工作

很多行业审查任务本质上是“按机构标准检查材料并输出结构化表格”。Claude Code 适合承担格式化、对齐、提取和初步整理，但专业标准必须来自行业团队自己。

## 核心概念

审查表、立场确认、结构化输出、业务解释、复核友好、专业标准外置

## 本案中的演示任务

使用 `verticals/lawyer/skills/lawyer-contract-review/` 作为课堂样例，围绕 SaaS 合同与补充协议演示 Skill 如何要求读取指定文件、按表格输出、标注来源和待律师确认事项。

## 推荐提示词

```text
请分析 lawyer-contract-review 这个教学 Skill 如何把“合同审查表”封装成可复用动作。请指出哪些字段是 Skill 结构，哪些字段必须由真实律师团队按自己的审查标准替换。
```

## 预期输出

合同审查 Skill 设计说明，包含输入材料、输出表格、立场确认、来源标注、人工复核字段和可替换区域。

## 学员如何替换成自己的 SOP

要求学员带入自己机构已有的合同审查模板，把“风险等级、审查维度、修改建议、业务解释”替换成自己的版本。课程只教如何封装，不定义审查标准。

## 课堂练习

让学员选一份非敏感合同模板，列出 5 个本机构常审字段，然后把这些字段改造成 Skill 输出表头和复核点。

## 检查点 / 常见误区

- 是否把示例风险等级当成通用法律标准。
- 是否没有确认服务方 / 客户方 / 中立视角。
- 是否输出了结论但没有来源文件。
- 是否让 Skill 直接替代律师审查。

## 讲课提示

强调 Skill 的价值是复用机构已有标准，而不是由课程发明行业标准。

## 本课交付物

一份合同审查 Skill 结构稿，以及本机构审查表替换清单。
