# 法律输出复核报告（教学示例）

> 本报告模拟 Risk Reviewer 视角，只负责发现问题，不替代律师最终判断。

| 问题清单 | 风险等级 | 涉及文件 | 修改建议 | 是否必须律师确认 |
|----------|----------|----------|----------|------------------|
| 交付邮件提到附件，但附件内容未见 | 高 | `交付确认邮件.md`、案件摘要、诉讼准备报告 | 不要直接写“已完整交付全部资料”，改为“已发送交付说明，附件内容待补充核验” | 是 |
| 违约金起算日尚未确认 | 高 | `软件服务合同.md`、`发票与付款记录.md`、诉讼准备报告 | 将违约金起算日标注【需律师确认】，补充尾款发票送达和付款条件成就证据 | 是 |
| “视为验收”是否成立需要结合异议时间和质量问题判断 | 高 | `软件服务合同.md`、`客户投诉邮件.md` | 避免直接写“B 公司已验收通过”，改为“可主张构成视为验收，需结合异议和使用情况论证” | 是 |
| 系统登录记录证明力需补强 | 中 | `系统登录记录.md`、证据目录 | 补充后台导出说明、账号对应关系、防篡改说明或公证方案 | 是 |
| 客户投诉邮件对我方并非完全有利 | 中 | `客户投诉邮件.md`、诉讼准备报告 | 在报告中单列不利事实和回应方案，避免只选择性引用 | 是 |
| 客户汇报邮件不能出现内部工具名称 | 低 | `06-客户汇报邮件.md` | 当前未出现内部 Agent 或 AI 提示词名称，保持客户可读表达 | 否 |
| 模拟材料中的邮箱为占位符 | 低 | `交付确认邮件.md`、`客户投诉邮件.md` | 课堂讲解时说明为模拟信息，不对应真实主体 | 否 |

## 总体复核意见

当前输出适合作为课堂演示和律师内部初稿。正式用于客户或诉讼前，必须补充原件、附件、送达凭证、系统日志原始导出和律师法律检索意见。不得将本示例直接作为正式法律意见或诉讼文件使用。
