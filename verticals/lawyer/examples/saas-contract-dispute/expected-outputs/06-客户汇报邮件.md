# 客户汇报邮件（教学示例）

> 客户版本示例。发送前需律师复核，不应包含 Claude、AI 提示词或内部 Agent 名称。

主题：关于软件服务合同尾款纠纷的初步梳理与后续材料补充建议

A 公司项目负责人您好：

我们已基于目前收到的合同、补充协议、交付沟通、付款记录、系统使用记录及对方律师函，对本案进行了初步梳理。现阶段初步判断，本案的核心在于证明贵司已按合同完成系统交付、B 公司已接收并持续使用系统，以及尾款支付条件已经成就。

目前较为关键的有利材料包括：

1. 合同中关于付款、验收及逾期未异议且继续使用视为验收的约定；
2. 2026 年 2 月 21 日的交付确认邮件及 B 公司“已收到”的回复；
3. 微信沟通中 B 公司确认业务部门已在使用系统、尾款流程需审批的记录；
4. 系统登录记录显示 B 公司多个账号在 3 月、4 月持续使用系统；
5. 首付款、尾款发票及尾款未支付记录。

同时，对方可能会围绕“系统未达到验收标准”“搜索准确率不足”“投诉后未完成修复”等方向提出抗辩。为增强我方证据链，建议尽快补充以下材料：

- 合同及补充协议的完整盖章扫描件；
- 交付邮件附件、系统访问地址、管理员账号交付记录；
- 培训会议记录、参会人员名单或录屏；
- 后台登录记录的原始导出文件及账号对应关系说明；
- 对客户投诉问题的处理记录、工单、优化说明或版本更新记录；
- 催款函送达凭证。

在上述材料补齐后，我们可进一步完善证据目录、诉讼请求和起诉材料。关于是否起诉、具体诉讼请求金额、违约金起算方式及管辖法院，仍需结合完整材料进行律师复核后确定。

以上为阶段性工作汇报，供您内部决策参考。
