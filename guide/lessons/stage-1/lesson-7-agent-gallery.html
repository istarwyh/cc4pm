子代理工作原理

Claude Code 的子代理在独立的上下文窗口中运行，完成后只将摘要返回给父级。

主对话上下文

什么时候该用子代理？

回答三个问题，决定是否需要启用子代理。

1 你的任务需要读取大量文件吗？
2 你需要基于之前的对话历史吗？
3 任务的输出比输入短得多吗？

代理 vs 技能

两种扩展机制，适用于不同场景。

🤖 Agent 代理

  • 独立上下文窗口，互不干扰
  • 隔离执行，有自己的工具权限
  • 适合专业任务（审查、构建修复等）
  • 输出摘要返回，节省主上下文空间
  • 可指定模型（opus / sonnet）
  • 用 YAML frontmatter 定义角色和工具

📚 Skill 技能

  • 加载到当前上下文，共享完整记忆
  • 无独立工具权限，依赖主对话
  • 适合领域知识和工作流指导
  • 通过 /skill 命令按需加载
  • 共享同一模型和对话历史
  • Markdown 格式，含使用场景说明
简单记忆法： Agent 是请一位专家来帮忙，Skill 是给自己一本参考手册。 需要独立执行和隔离时用 Agent，需要知识指导和上下文共享时用 Skill。
