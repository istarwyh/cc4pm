# Lesson 20.1: 26 个故事框架——从启蒙到冲突的完整工具箱

## 本课目标

- 掌握 CIS 故事讲述体系的 6 大类别、26 个框架
- 理解每个框架的结构、适用场景和实际示例
- 学会根据目的和受众快速选择最合适的故事框架

> **前置知识**：你已经在 Lesson 20 认识了 Sophia（故事讲述代理）和 Caravaggio（演示大师代理），并了解了故事讲述的心理学原理和 10 步工作流。本课是那 26 个框架的完整详解——是你的故事"字典"。

## 核心内容

### 框架分类总览

CIS 的故事讲述体系有 **26 个框架，分 6 个大类**：

| 类别 | 数量 | 核心目的 | 典型场景 |
|------|------|---------|---------|
| 启蒙故事 | 4 | 唤醒对问题/机会的意识 | 产品初期教育、宣传 |
| 转变故事 | 5 | 展示从旧状态到新状态的改变 | 案例研究、客户推荐 |
| 连接故事 | 5 | 建立情感连接 | 创始人自述、团队介绍 |
| 学习故事 | 4 | 传授智慧和经验 | 内部分享、教学内容 |
| 启发故事 | 4 | 激励行动 | 变革推动、产品发布 |
| 冲突故事 | 4 | 展示紧张和解决 | 大型演讲、市场营销 |

---

### 类别 1: 启蒙故事（Awakening Stories）— 4 个框架

这类故事的目的是**唤醒观众**对问题、机会或可能性的意识。

```
1. 问题启蒙（Problem Awakening）
   结构：无意识 → 发现问题 → 意识到严重性
   例："我以前以为记账只是为了报税。
       后来发现其实是管理现金流的关键。
       这改变了我对记账的看法。"

2. 机遇启蒙（Opportunity Awakening）
   结构：世界如此 → 如果可能会怎样 → 新可能性
   例："智能手机刚出现时，
       大多数人觉得是'奢侈品'。
       直到有人用 App 改变了日常生活。
       现在智能手机是必需品。"

3. 价值启蒙（Value Awakening）
   结构：旧价值观 → 体验新视角 → 价值观转变
   例："我以前觉得'便宜就是好'。
       直到用了一个高质量工具，
       省了我一个月的工作时间。
       现在我明白'省时间比省钱更值钱'。"

4. 能力启蒙（Capability Awakening）
   结构：我不会 → 有人展示 → 我发现我可以
   例："我以前觉得自己做不了财务分析。
       看了智能报表生成的分析对比，
       我发现其实很简单。"
```

**何时使用**：用户还不知道有这样的问题/机遇存在时。常见于产品初期的教育和宣传。

---

### 类别 2: 转变故事（Transformation Stories）— 5 个框架

这类故事的目的是**展示改变**——从旧状态到新状态。

```
5. 英雄之旅（Hero's Journey）
   结构：普通生活 → 面临挑战 → 寻求帮助 → 成功蜕变 → 回到新常态
   例："一个初创者，
       原本靠纸质账本管理。
       面临快速增长的挑战。
       用了 AI 记账工具（魔法助手）。
       3 个月内实现财务可视化。
       现在自信地规划下一步融资。"

6. 学徒故事（Apprenticeship Story）
   结构：我是初学者 → 学习过程 → 掌握技能 → 成为专家
   例："我是个 PM 新手，
       不知道怎么做需求分析。
       用了 BMAD 的 Epics 工作流。
       逐步学会了怎么拆解需求。
       现在能带团队了。"

7. 逆境故事（Adversity Story）
   结构：挑战 → 失败 → 坚持 → 最终突破
   例："创业初期月活只有 100 人，
       试了很多增长策略都没效果。
       直到用 Trigger Map 分析用户心理，
       发现关键驱动力被忽视了。
       改进后迎来爆发增长。"

8. 对比故事（Before-After Story）
   结构：用前的困境 → 使用产品 → 用后的改善
   例："用前：每月对账要花 5 天，经常错
       用后：1 天完成，零错误，多出 4 天做战略"

9. 合作故事（Collaboration Story）
   结构：各自为战 → 意识到需要合作 → 合作共赢
   例："PM 和设计师各做各的。
       后来在 WDS 的 Trigger Map 研讨会上合作。
       发现一起工作效率翻倍，设计质量也更好。"
```

**何时使用**：已经有用户了，你要展示用户的成功故事和产品的实际影响。常见于案例研究、客户推荐。

### 类别 3: 连接故事（Connection Stories）— 5 个框架

这类故事的目的是**建立情感连接**——让观众和主人公产生共鸣。

```
10. 同理心故事（Empathy Story）
    结构：我也经历过这个 → 我的感受 → 我怎么解决的
    例："我也是产品主理人，也经历过需求拆解的混乱。
        当时很焦虑，觉得自己不够专业。
        后来用了 BMM 的工作流，找到了解决办法。
        现在我想帮其他 PM 也找到这个办法。"

11. 家庭故事（Family/Community Story）
    结构：我们都是这个群体 → 我们共同的挑战 → 我们的团结
    例："作为小微企业主，
        我们都面临相同的问题：财务混乱。
        我创建了一个用户社区。
        大家在社区里互相帮助、分享经验。"

12. 成长故事（Growth Story）
    结构：我小的时候... → 我如何成长 → 现在的我...
    例："我刚入行时只会写代码。
        后来接触了产品思维，
        现在能独立做产品决策。"

13. 脆弱故事（Vulnerability Story）
    结构：我的失败/不安 → 我如何面对 → 教训
    例："我的第一个产品完全失败了。
        用户一个都没有。
        这次失败让我学会了用户研究的重要性。
        现在做的每个产品都先做 Trigger Map。"

14. 信仰故事（Belief Story）
    结构：我相信... → 为什么我相信 → 这个信念如何指导我
    例："我相信好的 AI 工具应该增强人的能力，而不是替代人。
        这个信念指导我设计的每个功能。
        比如 AI 分类后，还要让用户确认。"
```

**何时使用**：你要和观众建立个人连接、展示你的价值观和热情时。常见于创始人自述、团队介绍。

### 类别 4: 学习故事（Learning Stories）— 4 个框架

这类故事的目的是**传授智慧**——用过去的经验教导别人。

```
15. 失败教训故事（Failure Lesson Story）
    结构：我做错了... → 结果是... → 教训
    例："我第一次做 PRD，
        没有和工程师沟通，
        导致实现和需求完全不同。
        教训：PRD 要和所有利益相关者对齐。"

16. 对话故事（Dialogue Story）
    结构：两个人的对话 → 一个人的启发
    例：PM vs 设计师的故事
        PM："这个功能用户肯定需要"
        设计师："但我从 Trigger Map 看不到这个驱动力"
        PM："哦，我明白了。我们确实没有验证这个假设。"

17. 问题解决故事（Problem-Solving Story）
    结构：面临问题 → 思考过程 → 解决方案 → 结果
    例："需求变更很频繁，
        我想了想，引入了 BMM 的 Correct Course 工作流。
        现在变更有系统地管理，团队不再混乱。"

18. 实验故事（Experimentation Story）
    结构：假设 → 实验方法 → 发现 → 行动
    例："我假设用户最在乎快速记账。
        做了 A/B 测试：3 步 vs 5 步流程。
        发现 3 步的完成率反而低（用户害怕出错）。
        改变了我对'简化'的理解。"
```

**何时使用**：你要教导团队或用户最佳实践时。常见于内部分享、教学内容。

### 类别 5: 启发故事（Inspiration Stories）— 4 个框架

这类故事的目的是**激励行动**——让听众相信"我也可以做到"。

```
19. 梦想故事（Vision Story）
    结构：现在的困境 → 美好的未来想象 → 怎么到达那里
    例："现在产品主理人的工作混乱又低效。
        想象：如果有一套系统工作流，
        PM 可以高效地从想法到执行。
        这就是 cc4pm 要做的。"

20. 可能性故事（Possibility Story）
    结构：这曾经认为不可能 → 现在实现了
    例："曾经认为 AI 无法理解复杂的财务逻辑。
        现在有些 AI 工具能自动分类 95% 的交易。
        这打开了很多新可能。"

21. 鼓励故事（Encouragement Story）
    结构：如果这个人能做到，你也能
    例："一个 50 多岁的小店老板，
        以前从没用过 App。
        用了 AI 记账工具，现在每天都在用。
        这证明学习新工具永远不晚。"

22. 启蒙故事（Illumination Story）
    结构：我曾经不理解 → 一个时刻改变了我 → 现在我理解了
    例："我以前不理解为什么要做 Trigger Map。
        第一次看到它如何引导设计决策时，
        我明白了——这是从用户心理出发的科学方法。"
```

**何时使用**：你要激励团队采取行动、相信新方法时。常见于变革推动、产品发布演讲。

---

### 类别 6: 冲突故事（Conflict Stories）— 4 个框架

这类故事的目的是**展示紧张和解决**——让观众跟随情节。

```
23. 对立故事（Clash Story）
    结构：两种思想的冲突 → 哪个更对 → 发现第三条路
    例："功能导向 vs 设计导向
        两个派系一直在争论。
        直到用 Trigger Map 整合两个视角。
        发现'从用户心理出发'能同时满足两个需求。"

24. 悖论故事（Paradox Story）
    结构：看似矛盾的两个真实 → 如何共存
    例："好的设计通常很复杂。
        但好的产品应该简单。
        这看似矛盾。
        但 WDS 的'阳光路径'原则解决了这个悖论。"

25. 讽刺故事（Irony Story）
    结构：预期的结果 vs 实际的结果
    例："我们以为功能越多用户越高兴。
        结果发现功能太多反而导致用户流失。
        从 Trigger Map 发现，用户最想要的是'简单'。"

26. 救赎故事（Redemption Story）
    结构：失败/坏事 → 看起来无法挽回 → 意外的转机 → 新的开始
    例："公司面临倒闭，所有投资者都跑了。
        最后一个忠诚的用户说'你们的产品改变了我的生意'。
        这句话给了创始人继续的勇气。
        现在公司扭亏为盈。"
```

**何时使用**：你要制造悬念、引发思考、让观众记住核心观点时。常见于大型演讲、市场营销。

### 框架选择速查

| 你的目的 | 推荐类别 | 最常用框架 |
|---------|---------|-----------|
| 让人意识到问题存在 | 启蒙故事 | 问题启蒙、机遇启蒙 |
| 展示产品实际效果 | 转变故事 | 对比故事、英雄之旅 |
| 和观众建立个人连接 | 连接故事 | 同理心故事、脆弱故事 |
| 教导最佳实践 | 学习故事 | 问题解决故事、失败教训 |
| 激励团队采取行动 | 启发故事 | 梦想故事、鼓励故事 |
| 制造悬念引发思考 | 冲突故事 | 悖论故事、讽刺故事 |

**先学 5 个就够用**：英雄之旅、对比故事、失败教训、同理心故事、转变故事。需要时查阅完整列表。

## 常见问题

**Q: 26 个框架是不是都要学？**

A: 不需要全部掌握。先学最常用的 5 个（英雄之旅、对比故事、失败教训、同理心故事、转变故事），覆盖 80% 的场景。其他框架在需要时查阅即可。

**Q: 同一个故事可以用不同框架吗？**

A: 可以。同一个用户经历，用"对比故事"强调用前用后的差异，用"英雄之旅"强调成长蜕变，用"同理心故事"强调情感共鸣。选择取决于你的受众和目的。

**Q: 这些框架可以组合使用吗？**

A: 可以。比如一个长演示中，开场用"问题启蒙"引发关注，中段用"对比故事"展示价值，结尾用"梦想故事"激励行动。
