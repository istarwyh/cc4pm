### 从"设计"到"叙事"——为什么需要故事讲述

你已经完成了 WDS 的核心工作：

```
Trigger Map    → 理解用户"为什么"
UX 场景        → 定义用户"怎么用"
UX 设计        → 画出用户"看到什么"
```

但这还不够。还有一个关键问题没有回答：**"我怎么让用户和投资者相信这个产品价值？"**

这就是故事讲述的价值：

```
设计好的产品    ← 只有专业人士能理解
    ↓
通过故事讲述    ← 连非专业人士也能理解和被说服
    ↓
产品演示/推介   ← 用户、投资者、合作伙伴都买单
```

**一个真实例子**：

```
设计师说："我们优化了支付流程，从 7 步减到 3 步。"
用户听了不感动。

但如果这样说：
"一个外卖员，骑了 30 分钟到了地点。
因为支付太复杂，他放弃了，跑去隔壁竞品点餐。
我们的优化让他在 15 秒内完成支付，
赚到这笔配送费，回家按时吃饭。"

这个故事 30 秒就让人理解了产品的真正价值。
```

### 故事讲述的核心原理

为什么故事这么有力？有四个心理学原理：

#### 1. 镜像神经元（Mirror Neurons）

当人听故事时，大脑会"模拟"故事中的场景和情感。

```
数据："转化率提升 12%"           → 大脑：记一下
故事："用户等了 15 秒，最后还是点了返回..." → 大脑：我感受到了焦虑和等待
```

#### 2. 记忆粘性（Stickiness）

```
数据："45% → 50%"  → 听过就忘
故事："一个用户从放弃到成功的转变"  → 能记一辈子
```

#### 3. 说服力倍增

```
论证："我们的产品很好"  → 听众："你当然这么说"
故事："一个用户这样用了，结果..."  → 听众自己得出结论
```

#### 4. 情感共鸣（Emotional Resonance）

```
功能说明："支持 API 调用"  → 技术人员关心
故事："一个开发者用 API 3 天搭了整个系统"  → 所有人都能体会价值
```

### WDS 中的故事讲述

在 WDS 的框架中，故事讲述出现在两个关键时刻：

```
Phase 2: Trigger Map 研讨会
         ↓ 讲故事帮助团队理解用户心理驱动力
         例："小微企业主张先生的故事"

Phase 3-4: 设计和演示
         ↓ 讲故事帮助用户和投资者理解产品价值
         例："张先生从焦虑到安心的故事"
```

特别是在 Phase 3（UX 场景）中，每个场景本质上就是一个"用户故事"——有人物、情境、冲突、解决方案、结果。

### CIS 故事讲述的 10 步工作流

现在你知道了故事的原理和框架。但"知道框架"和"能讲好故事"是两回事。CIS 提供了一个 10 步工作流，帮你从选择框架到讲好故事。

#### 阶段 1：规划（Steps 1-3）

```
Step 1: 明确目的    → 唤醒认识？展示价值？建立连接？激励行动？
Step 2: 选择框架    → 从 26 个框架中选（见 Lesson 20.1）
Step 3: 核心事件    → 这个框架在你的经历中对应什么事件？
```

#### 阶段 2：构思（Steps 4-6）

```
Step 4: 定义角色    → 主人公是谁？背景、性格、目标
Step 5: 勾勒情节    → 开场 → 冲突 → 转折 → 高潮 → 结局
Step 6: 核心洞察    → 用一句话表达故事的核心观点
```

#### 阶段 3：细节（Steps 7-9）

```
Step 7: 感官细节    → 声音、颜色、气味——让故事变成画面
Step 8: 编写对话    → 好的对话推动情节，节省篇幅
Step 9: 镜头切换    → 哪里停顿、哪里加重音，提高节奏感
```

#### 阶段 4：交付（Step 10）

```
Step 10: 试讲调整   → 小组试讲 → 收集反馈 → 调整
```

### 从 UX 场景到故事讲述

你之前在 Lesson 19 创建的 UX 场景，本质上已经是故事框架了。现在你可以用这些场景来创作市场营销和用户教育的故事。

**转化逻辑**：

```
UX 场景：
  用户：小微企业主张先生，在便利店买了原材料
  驱动力：害怕忘记这笔现金支出
  最短路径：打开App → 拍发票 → AI分类 → 确认

故事化后（用"对比故事"框架）：
  "张先生做早餐生意。

  用前：每天都怕漏记现金。
  最害怕的时刻是月末发现对不上账，
  整晚失眠查账。

  用后：打开App拍一张发票，
  AI自动分类，3秒完成。
  现在他再也不会因为漏账而焦虑。

  简单一个动作，改变了他每个月一次的焦虑。"
```

这个故事不仅让潜在用户理解产品，也让投资者看到真实的用户价值。

### 故事和 WDS 的关联

故事讲述不是独立存在的——它和前面学的 WDS 紧密相连：

```
Trigger Map 中的"驱动力"
    ↓ 变成故事中的"情感共鸣点"
    用户的恐惧和渴望 → 故事的冲突和希望

UX 场景中的"用户旅程"
    ↓ 变成故事中的"案例和演示"
    张先生拍照记账的场景 → 路演中的产品演示

功能影响分析（FIA）的"数据"
    ↓ 变成故事中的"数据支撑"
    功能覆盖了 6 个核心驱动力 → "我们理解用户的 6 个核心需求"
```

**核心思想**：你不是在"编故事"，你是在把 Trigger Map 和 UX 场景中发现的真实洞察，用故事的形式传达给不同的受众。

## 常见问题

**Q: 故事讲述和营销文案有什么区别？**

A: 文案是"说服"，故事是"感动"。文案告诉你"买这个产品"，故事让你"想成为这样的人"。故事更深层，因为它改变认知而不仅仅是提供信息。好的营销同时包含两者——故事做深度，文案做清晰。

**Q: 我没有用户故事怎么办？**

A: 可以虚构（注明是代表性场景）。但更好的做法是：(1) 用你自己作为第一个用户讲故事；(2) 采访早期用户提取他们的故事；(3) 用 Trigger Map 和 UX 场景虚拟构造一个故事，这个故事代表了研究发现。

**Q: 26 个框架和 7 种演示类型在哪里看？**

A: 详见补充课 Lesson 20.1（26 个故事框架）和 Lesson 20.2（7 种产品演示类型）。本课聚焦原理和工作流，补充课提供完整的工具箱。
