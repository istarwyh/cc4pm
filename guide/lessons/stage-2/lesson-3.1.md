# Lesson 13.1: 网站情报工具——竞品分析的实战利器

## 本课目标

- 掌握网站情报工具的核心能力：流量分析、SEO 体检、技术栈检测、变现追踪
- 学会使用 WhatsTheWeb 等浏览器扩展快速获取竞品情报
- 建立"工具辅助决策"的竞品分析工作流

## 核心内容

### 为什么需要网站情报工具

在 Lesson 13 中，Mary 帮你完成了宏观的市场研究。但当你需要**快速了解一个具体网站**时——比如竞品的流量规模、用了什么技术、SEO 做得怎么样——你需要的是"网站情报工具"。

这类工具的核心价值：**把分散的站点信息整合到一个视图里，让你在几秒内建立对一个网站的全面认知**。

### 四大核心能力

#### 1. 流量分析

**你能看到什么**：
- 月访问量估算（全球排名、国家排名）
- 流量来源渠道（直接访问、搜索、社交、外链、邮件）
- AI 引流分析（来自 ChatGPT、Perplexity 等 AI 工具的流量）
- 流量趋势变化

**PM 怎么用**：
- 评估竞品的市场体量：月访问 100 万 vs 10 万，策略完全不同
- 发现流量渠道机会：竞品靠 SEO 引流，你可以打社交媒体差异化
- 追踪 AI 流量趋势：越来越多用户通过 AI 工具发现产品

**数据来源**：多数工具基于 SimilarWeb、Semrush 等第三方估算数据，不是精确值，但足够用于竞争态势判断。

#### 2. SEO 体检

**你能看到什么**：
- 页面标题（title）和描述（description）是否规范
- H 标签层级结构是否合理
- canonical 标签设置
- robots.txt 和 sitemap.xml 状态
- 图片 alt 属性覆盖率
- 内链和外链结构

**PM 怎么用**：
- 快速判断竞品的 SEO 投入程度：标题写得好、sitemap 完整 = 认真做 SEO
- 发现竞品的 SEO 薄弱点：图片没 alt、H 标签混乱 = 你可以用 SEO 超越
- 学习竞品的 SEO 策略：看他们怎么写标题、怎么布局关键词

#### 3. 技术栈检测

**你能看到什么**：
- 前端框架（React、Vue、Angular、Svelte）
- CMS 平台（WordPress、Webflow、Shopify）
- 分析工具（Google Analytics、Mixpanel、Amplitude）
- CDN 和托管服务
- 其他前端库和工具

**PM 怎么用**：
- 技术选型参考：竞品用 React + Vercel，说明这个组合经过验证
- 评估技术壁垒：竞品用 WordPress = 技术门槛低，容易复制
- 了解行业标配：如果所有竞品都用 Google Analytics，这是行业默认选择

#### 4. 变现追踪（AdSense 检测）

**你能看到什么**：
- 是否接入 Google AdSense
- Publisher ID 是什么
- 同一 Publisher ID 下还有哪些域名

**PM 怎么用**：
- 了解竞品的变现模式：接广告 = 流量变现路线
- 发现关联站点：同一 Publisher ID 的其他域名可能是竞品矩阵
- 评估广告密度：广告太多 = 用户体验差，可能是你的差异化机会

### 实战工具推荐

#### WhatsTheWeb

**定位**：轻量级侧边栏扩展，一键查看网站情报

**核心特点**：
- 侧边栏模式：不切换标签页，直接在当前页面查看
- 四合一整合：流量、SEO、技术栈、AdSense 一个面板搞定
- 免费使用

**安装**：[Chrome Web Store](https://chromewebstore.google.com/detail/whatstheweb-website-traff/dmdbpmhalempefopiaiphpnohhmahgmi)

**使用方式**：
1. 安装扩展
2. 访问任意网站
3. 点击浏览器工具栏的 WhatsTheWeb 图标
4. 侧边栏滑出，展示该站点的完整情报

**适合场景**：日常浏览时随手查看竞品信息，快速建立认知

### 竞品分析工作流

将网站情报工具融入你的竞品分析流程：

```
Step 1: 列出竞品清单
        ↓ （Mary 的市场研究帮你完成）
Step 2: 逐一扫描竞品网站
        ↓ （WhatsTheWeb / Wappalyzer）
Step 3: 填充竞品对标矩阵
        ↓ （Lesson 13 的竞品对标模板）
Step 4: 发现差异化机会
        ↓ （流量渠道、技术选型、SEO 策略）
Step 5: 转化为产品决策
        ↓ （Lesson 13 的决策输出模板）
```

**实操示例**：假设你在做一款在线设计工具，需要分析竞品：

```bash
# 1. 访问竞品网站
# 2. 点击 WhatsTheWeb 图标
# 3. 记录关键信息：
```

```markdown
## 竞品情报卡：Figma

### 流量
- 月访问量：~100M
- 主要来源：直接访问（40%）、搜索（35%）、外链（15%）
- AI 引流：有增长趋势

### 技术栈
- 前端：React
- 分析：Google Analytics
- CDN：Cloudflare

### SEO
- 标题：规范，含关键词
- 描述：完整
- sitemap：有

### 变现
- AdSense：无（SaaS 订阅模式）

### 关键洞察
- 流量主要靠品牌直接访问 = 品牌力强
- SEO 投入中等 = 有超越空间
- 技术栈主流 = 技术壁垒不在框架选择
```

## 常见问题

**Q: 流量数据准确吗？**

A: 不精确，但足够用于竞争态势判断。这些数据基于第三方估算（SimilarWeb 等），与实际可能有 20-50% 的偏差。重要的是相对比较——如果竞品 A 的流量是竞品 B 的 3 倍，这个量级关系是可信的。

**Q: 技术栈检测能检测到后端技术吗？**

A: 有限。浏览器扩展主要能检测前端可见的技术（框架、CDN、分析工具）。后端语言（Python、Go、Java）和数据库通常检测不到，除非网站使用了特定的响应头或错误页面暴露了信息。

**Q: 免费工具够用吗？**

A: 对于日常竞品监控足够。如果你需要精确的流量数据做商业决策（如投资评估），建议升级到 SimilarWeb 或 SEMrush 的付费版本。

**Q: 这些工具会侵犯隐私吗？**

A: 这类工具分析的是公开可访问的网页信息，不涉及用户个人数据。但部分扩展会收集你的浏览历史，安装前建议查看隐私政策。
